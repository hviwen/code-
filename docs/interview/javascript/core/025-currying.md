# 025. [中级] 如何实现函数的柯里化（currying）？

> 来源：`docs/javascript/js_interview_questions_part_1.md`

## 问题本质解读

柯里化是把一个接收多个参数的函数，转换成可以分多次接收参数的函数。它的核心不是“函数套函数”，而是延迟收集参数，直到参数足够再执行原函数。

一句话答法：柯里化让 `fn(a, b, c)` 变成 `fn(a)(b)(c)` 或 `fn(a, b)(c)`，适合预置参数和函数组合。

## 问题意图

这道题想确认你是否理解高阶函数、闭包、参数收集和延迟执行，也能写出一个能处理分批传参的简化实现。

## 考察范围

- 高阶函数和闭包。
- 参数收集与函数 `length`。
- 固定参数个数的柯里化实现。
- 柯里化、偏函数、bind 预置参数的区别。
- 表单校验、事件处理、配置函数、函数组合等场景。

## 技术错误纠正

原始内容方向正确，但容易把柯里化和“任意函数嵌套”混在一起。准确说法是：柯里化必须围绕原函数参数数量进行收集，参数数量满足后才执行原函数。

## 知识点系统梳理

核心过程：

```text
接收一部分参数 -> 不够就返回新函数继续收集 -> 够了就执行原函数
```

简化实现：

```js
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args)
    }

    return function next(...nextArgs) {
      return curried.apply(this, args.concat(nextArgs))
    }
  }
}
```

注意：这个实现依赖 `fn.length`，所以默认参数和剩余参数会影响判断。真实工具库会处理更多边界。

## 实战应用举例

### 示例 1：分批传参

```js
function add(a, b, c) {
  return a + b + c
}

const curriedAdd = curry(add)

console.log(curriedAdd(1)(2)(3)) // 6
console.log(curriedAdd(1, 2)(3)) // 6
console.log(curriedAdd(1)(2, 3)) // 6
```

这个例子证明：柯里化的关键是参数可以分批传入。

### 示例 2：预置校验规则

```js
const matchRule = curry((rule, value) => rule.test(value))

const isEmail = matchRule(/^[^@]+@[^@]+\.[^@]+$/)
const isPhone = matchRule(/^1\d{10}$/)

console.log(isEmail('a@test.com')) // true
console.log(isPhone('13800138000')) // true
```

这里先固定规则，再复用到多个值上，适合表单校验、权限判断和参数格式化。

## 使用场景说明和对比

| 技术 | 做什么 | 适合场景 |
| --- | --- | --- |
| 柯里化 | 多参数函数变成分批传参 | 函数组合、配置预置 |
| 偏函数 | 固定一部分参数，返回新函数 | 固定 API 前缀、固定规则 |
| `bind` | 固定 `this` 和前置参数 | 事件回调、class 方法 |
| 普通函数 | 一次性传入所有参数 | 简单业务逻辑 |

不建议使用柯里化的场景：

- 团队不熟悉函数式写法，读起来比原函数更绕。
- 参数很多且含义不清晰，应该改成对象参数。
- 只是为了炫技，没有复用配置或组合需求。

## 易错点提示

- 柯里化不是把函数随便嵌套几层，而是按参数数量收集参数。
- `fn.length` 不统计默认参数后的参数，也不统计剩余参数。
- 柯里化会创建额外函数，热路径里不要滥用。
- 多参数都属于同一个业务对象时，对象参数比柯里化更清晰。
- `this` 传递容易被忽略，简化实现里要用 `apply` 保留调用上下文。

## 记忆要点总结

- 柯里化 = 多参数函数分批收参。
- 实现核心：闭包保存已收集参数。
- 参数够了执行原函数，不够继续返回函数。
- 价值：预置配置、函数复用、函数组合。
- 风险：过度函数式会降低可读性。

## 可能类似的问题及简要参考答案

**Q：柯里化依赖闭包吗？**  
A：依赖。闭包用于保存前几次调用已经传入的参数。

**Q：柯里化和 `bind` 一样吗？**  
A：不一样。`bind` 主要绑定 `this` 并可预置参数；柯里化强调把多参数调用拆成连续的函数调用。

**Q：柯里化有什么实际价值？**  
A：可以预置规则或配置，生成更具体的函数，减少重复参数并方便函数组合。

## 辅助记忆总结

记成一句话：柯里化就是“参数不够先攒着，攒够了再执行”。
