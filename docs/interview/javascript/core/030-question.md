# 030. [高级] 解释函数式编程中的纯函数概念

> 来源：`docs/javascript/js_interview_questions_part_1.md`

## 问题本质解读

纯函数指相同输入永远得到相同输出，并且执行过程中不产生副作用。它的价值是让代码可预测、可测试、可缓存，也更适合组合。

一句话答法：纯函数只依赖输入，不修改外部世界。

## 问题意图

这道题想确认你是否理解函数式编程的基本约束，并能把纯函数用于前端状态更新、数据转换和单元测试，而不是只背“无副作用”。

## 考察范围

- 相同输入相同输出。
- 副作用的定义。
- 不修改外部变量、不修改入参。
- 纯函数在数组转换、状态管理、reducer 中的应用。
- 纯函数与缓存、测试、并发的关系。
- 纯函数和非纯函数的边界。

## 技术错误纠正

原始内容中“修改其他变量值”应明确为“不修改外部状态，也不修改传入对象本身”。同时还要补充：读取时间、随机数、网络请求、DOM 操作、日志输出都可能让函数不纯。

## 知识点系统梳理

纯函数满足两个条件：

1. 相同输入总是返回相同输出。
2. 不产生副作用。

副作用包括：

| 副作用 | 例子 |
| --- | --- |
| 修改外部变量 | `count += 1` |
| 修改入参 | `user.name = 'x'` |
| I/O 操作 | 请求接口、读写文件、读写 localStorage |
| DOM 操作 | `document.body.append(...)` |
| 不确定输入 | `Date.now()`、`Math.random()` |

纯函数不是要求项目里所有函数都纯，而是把可纯的计算逻辑尽量做纯，把副作用集中到边界。

## 实战应用举例

### 示例 1：状态更新保持纯

```js
function addTodo(todos, text) {
  return todos.concat({
    id: todos.length + 1,
    text,
    done: false,
  })
}

const oldTodos = []
const newTodos = addTodo(oldTodos, 'learn closure')

console.log(oldTodos.length) // 0
console.log(newTodos.length) // 1
```

这个例子证明：纯函数不修改旧状态，而是返回新状态。这是 Redux reducer、React 状态更新常见思路。

### 示例 2：把副作用放到外层

```js
function formatUser(user) {
  return `${user.name} (${user.role})`
}

async function renderUser(id) {
  const user = await fetchUser(id)
  document.querySelector('#user').textContent = formatUser(user)
}
```

`formatUser` 是纯函数，容易测试；请求和 DOM 更新是副作用，集中放在 `renderUser` 里。

## 使用场景说明和对比

| 场景 | 适合纯函数吗 | 原因 |
| --- | --- | --- |
| 数据格式化 | 适合 | 输入输出明确 |
| 表单校验 | 适合 | 同样输入应有同样校验结果 |
| reducer/state 更新 | 适合 | 方便回放、测试、比较 |
| 请求接口 | 不纯 | 依赖外部系统 |
| DOM 操作 | 不纯 | 修改外部环境 |
| 读取当前时间 | 不纯 | 每次调用结果可能不同 |

纯函数和非纯函数的协作方式：把计算逻辑写成纯函数，把请求、缓存、日志、DOM 等副作用放在边界层。

## 易错点提示

- 不修改外部变量还不够，也不能修改传入对象。
- `array.sort()` 会原地修改数组，不是纯操作；可先复制再排序。
- `Date.now()`、`Math.random()` 会让相同输入产生不同输出。
- `console.log` 也是副作用，只是通常影响较小。
- 纯函数不代表没有业务价值；复杂业务规则反而更适合抽成纯函数测试。

## 记忆要点总结

- 纯函数 = 相同输入相同输出 + 无副作用。
- 好处：可预测、可测试、可缓存、可组合。
- 前端常见：数据转换、表单校验、reducer。
- 不纯边界：请求、DOM、时间、随机数、日志。
- 项目目标是隔离副作用，不是消灭副作用。

## 可能类似的问题及简要参考答案

**Q：纯函数有什么好处？**  
A：可预测、容易测试、容易缓存，也更适合函数组合。

**Q：读取 `Date.now()` 的函数是纯函数吗？**  
A：通常不是。即使参数相同，每次调用也可能得到不同结果。

**Q：如何把非纯逻辑拆开？**  
A：把请求、DOM、时间等副作用放在外层，把数据计算和格式化抽成纯函数。

## 辅助记忆总结

记成一句话：纯函数就是“不偷看外面，也不改外面”。
