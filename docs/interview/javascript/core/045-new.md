# 045. [高级] 如何实现一个简单的 `new` 操作符？

> 来源：`docs/javascript/js_interview_questions_part_1.md`

## 问题本质解读

手写 `new` 的核心是复现它的四步：创建对象、连接原型、绑定 `this` 执行构造函数、按返回值规则决定最终结果。

一句话答法：`new` 会创建一个以构造函数 `prototype` 为原型的新对象，再用这个对象作为 `this` 调用构造函数。

## 问题意图

这道题考察你是否真正理解构造函数、原型链、`this` 绑定和构造函数返回值规则。

## 考察范围

- `Object.create(Constructor.prototype)`。
- `apply` 绑定构造函数的 `this`。
- 构造函数返回对象时覆盖默认实例。
- 构造函数返回基本类型时忽略。
- `prototype` 非对象时的边界。
- `new.target` 和 class 的限制。

## 技术错误纠正

实现 `new` 不能只做 `Ctor.apply(obj, args)`，还必须把新对象的原型连接到 `Ctor.prototype`，并处理构造函数显式返回对象的情况。

## 知识点系统梳理

`new Foo(a, b)` 的简化流程：

1. 创建新对象。
2. 新对象的 `[[Prototype]]` 指向 `Foo.prototype`。
3. 以新对象作为 `this` 执行 `Foo(a, b)`。
4. 如果 `Foo` 返回对象或函数，则返回该返回值；否则返回新对象。

## 实战应用举例

### 示例 1：简化版 myNew

```js
function myNew(Constructor, ...args) {
  if (typeof Constructor !== 'function') {
    throw new TypeError('Constructor must be a function')
  }

  const prototype =
    Constructor.prototype && typeof Constructor.prototype === 'object'
      ? Constructor.prototype
      : Object.prototype

  const instance = Object.create(prototype)
  const result = Constructor.apply(instance, args)

  const isObject = result !== null && (typeof result === 'object' || typeof result === 'function')
  return isObject ? result : instance
}
```

### 示例 2：验证返回值规则

```js
function User(name) {
  this.name = name
}

function Override() {
  this.name = 'inner'
  return { name: 'outer' }
}

console.log(myNew(User, 'Ada').name) // Ada
console.log(myNew(Override).name) // outer
```

这个例子证明：返回对象会覆盖默认实例，普通返回值不会。

## 使用场景说明和对比

| 点 | `new` | `myNew` 简化实现 |
| --- | --- | --- |
| 创建实例 | 引擎内置 | `Object.create` |
| 原型连接 | 自动 | 手动指定 prototype |
| 绑定 this | 自动 | `apply` |
| 返回值规则 | 自动 | 手动判断对象/函数 |
| class 支持 | 支持 | 不能直接 `apply` class |

这个手写题用于理解机制，不建议在业务里替代原生 `new`。

## 易错点提示

- `Object.create(Constructor.prototype)` 是连接原型的关键。
- 构造函数返回对象或函数时，要返回该对象。
- 构造函数返回基本类型时，要忽略它。
- 箭头函数没有 `prototype`，不能作为构造函数。
- ES6 class 不能被 `apply` 调用，所以简化版 `myNew` 不支持 class。

## 记忆要点总结

- 手写 `new` 四步：建对象、连原型、绑 this、判返回。
- 原型来自构造函数的 `prototype`。
- 返回对象覆盖实例。
- 返回基本类型忽略。
- 这是机制题，不是业务工具函数。

## 可能类似的问题及简要参考答案

**Q：手写 `new` 为什么要用 `Object.create`？**  
A：为了创建一个原型指向 `Constructor.prototype` 的新对象。

**Q：构造函数返回函数时怎么办？**  
A：函数也是对象，应返回该函数，而不是默认实例。

**Q：`myNew` 能完全模拟原生 `new` 吗？**  
A：不能。简化实现无法完整支持 class、`new.target` 等引擎级行为。

## 辅助记忆总结

记成一句话：手写 `new` 就是“造壳、接祖先、借 this 执行、看返回值”。
