# 044. [中级] `new` 操作符的执行过程是什么？

> 来源：`docs/javascript/js_interview_questions_part_1.md`

## 问题本质解读

`new` 是对象实例化流程的触发器。它会创建新对象、连接原型、绑定 `this` 执行构造函数，并根据构造函数返回值决定最终结果。

一句话答法：`new` 做四件事：造对象、接原型、绑定 this、判返回值。

## 问题意图

这道题考察构造函数、原型链、`this` 和返回值规则。能讲清 `new`，基本就能串起 JavaScript 面向对象的核心机制。

## 考察范围

- `new` 的四步执行过程。
- 构造函数的 `prototype`。
- 实例的内部原型。
- 构造函数里的 `this`。
- 返回对象和返回基本类型的差异。
- 箭头函数和 class 的构造限制。

## 技术错误纠正

`new` 不只是“执行构造函数”。执行构造函数之前，它已经创建了实例并连接了原型；执行之后，还要根据返回值决定返回什么。

## 知识点系统梳理

`new User('Ada')` 可以理解成：

```js
const obj = Object.create(User.prototype)
const result = User.call(obj, 'Ada')
return isObject(result) ? result : obj
```

正式步骤：

1. 创建一个空对象。
2. 将空对象的 `[[Prototype]]` 指向构造函数的 `prototype`。
3. 以新对象作为 `this` 执行构造函数。
4. 如果构造函数返回对象或函数，则返回该值；否则返回新对象。

## 实战应用举例

### 示例 1：this 指向新实例

```js
function User(name) {
  this.name = name
}

User.prototype.sayName = function () {
  return this.name
}

const user = new User('Ada')

console.log(user.name) // Ada
console.log(user.sayName()) // Ada
console.log(Object.getPrototypeOf(user) === User.prototype) // true
```

这个例子证明：`new` 既绑定了 `this`，也连接了原型。

### 示例 2：返回值规则

```js
function A() {
  this.name = 'A'
  return 1
}

function B() {
  this.name = 'B'
  return { name: 'override' }
}

console.log(new A().name) // A
console.log(new B().name) // override
```

构造函数返回基本类型会被忽略；返回对象会覆盖默认实例。

## 使用场景说明和对比

| 写法 | 是否触发 new 流程 | 结果 |
| --- | --- | --- |
| `User('Ada')` | 否 | 普通函数调用 |
| `new User('Ada')` | 是 | 创建实例 |
| `Reflect.construct(User, ['Ada'])` | 是 | 更底层的构造调用 |
| `User.call(obj, 'Ada')` | 否 | 只绑定 this，不连接原型 |

## 易错点提示

- 忘记 `new` 时，构造函数就是普通函数。
- 箭头函数不能被 `new` 调用。
- class 必须用 `new` 调用。
- 构造函数返回对象会覆盖默认实例。
- 实例方法通常应放在 `prototype` 上，避免每个实例重复创建。

## 记忆要点总结

- `new` 四步：建、连、绑、返。
- 原型来自构造函数的 `prototype`。
- 构造函数里的 `this` 是新对象。
- 返回对象覆盖实例，返回基本类型忽略。
- class 也依赖构造流程，只是语法更严格。

## 可能类似的问题及简要参考答案

**Q：`new` 时实例的原型是谁？**  
A：实例的内部原型指向构造函数的 `prototype`。

**Q：构造函数返回 `null` 会怎样？**  
A：`null` 不是可用的返回对象，通常会忽略并返回新实例。

**Q：`new` 能改变构造函数内部的 `this` 吗？**  
A：会。构造函数执行时，`this` 指向新创建的实例。

## 辅助记忆总结

记成一句话：`new` 是“先造空壳，再接原型，借壳执行，最后看返回”。
