# 016. [初级]** 函数声明和函数表达式有什么区别？

> 来源：`docs/javascript/js_interview_questions_part_1.md`

## 问题本质解读

这道题考察JavaScript中两种函数定义方式的差异，面试官想了解你是否理解提升机制和作用域的区别。

## 知识点系统梳理

函数声明:
```javascript
function fun(){ return ‘fun’ };
```
- 函数提升；
- 块级作用域；

函数表达式：
```javascript
const fun = function(name){ return name};
```
- 变量提升；
- 命名函数表达式；
- 适合作为参数传递；

### 知识点系统梳理

**函数声明特点：**
- 完整提升（包括函数体）
- 可以在声明前调用
- 在块级作用域中有特殊提升行为

**函数表达式特点：**
- 变量提升，但值为undefined
- 不能在赋值前调用
- 支持匿名函数和命名函数表达式

### 实战应用举例
```javascript
// 1. 提升行为对比
console.log(declared());    // "I'm declared" - 可以调用
// console.log(expressed()); // TypeError: expressed is not a function

function declared() {
  return "I'm declared";
}

var expressed = function() {
  return "I'm expressed";
};

console.log(expressed());   // "I'm expressed" - 现在可以调用
```

```javascript
// 2. 条件性函数定义的最佳实践
// ❌ 避免：条件性函数声明（行为不一致）
if (true) {
  function conditionalFunc() {
    return 'declared';
  }
}

// ✅ 推荐：使用函数表达式
let conditionalFunc;
if (true) {
  conditionalFunc = function() {
    return 'expressed';
  };
}
```

### 记忆要点总结
- **函数声明**：完整提升，可在声明前调用，有函数名
- **函数表达式**：变量提升但值为undefined，不能提前调用
- **使用场景**：函数声明用于主要功能，函数表达式用于条件性定义
- **最佳实践**：避免条件性函数声明，优先使用函数表达式

## 使用场景说明和对比

| 场景 | 函数声明 | 函数表达式 | 箭头函数 |
|------|---------|-----------|---------|
| 全局/模块作用域 | ✅ 推荐，提升后可任意位置调用 | ✅ 可选 | ✅ 需维护词法this |
| 事件回调 / setTimeout | ❌ 不能作为回调直接声明 | ✅ 匿名表达式 | ✅ 推荐，this指向外层 |
| 构造函数 | ✅ 可作构造函数 | ✅ 可作构造函数 | ❌ 没有[[Construct]] |
| 对象方法 | ❌ 提升可能导致引用错误 | ✅ 推荐 | ❌ this指向外层不绑定实例 |
| 块级条件分支 | ⚠️ ES6严格模式下报错 | ✅ 安全 | ✅ 安全 |

## 易错点提示

1. 函数声明会整体提升，函数表达式只会提升变量声明，赋值仍在原位置。
2. `var fn = function () {}` 在赋值前调用通常是 `TypeError`，不是 `ReferenceError`，因为变量已经存在但值是 `undefined`。
3. 块级作用域里的函数声明在非严格模式和旧浏览器中存在兼容差异，面试回答时不要把它当成稳定写法。
4. 命名函数表达式的函数名主要用于递归和调试栈，外层作用域通常访问不到这个内部名字。
5. 箭头函数是函数表达式的一种，但没有自己的 `this`、`arguments`，也不能作为构造函数。

## 记忆要点总结

- 函数声明：有名字、整体提升、适合模块级工具函数。
- 函数表达式：赋值后可用、可匿名或具名、适合回调和条件赋值。
- 箭头函数：更适合短回调，不适合需要动态 `this` 或构造实例的场景。
- 面试先答提升差异，再补作用域、this 和使用场景。

## 可能类似的问题及简要参考答案

**Q：函数表达式能不能在定义前调用？**  
A：不能。`var` 声明的函数表达式在赋值前是 `undefined`，调用会报 `TypeError`；`let/const` 声明在初始化前处于 TDZ，会报 `ReferenceError`。

**Q：什么时候优先用函数声明？**  
A：模块级、工具型、需要提前调用或递归的函数可以用函数声明；回调、条件分支和需要保留局部状态的场景更适合函数表达式。

## 辅助记忆总结

一句话记：声明是“先登记整个人”，表达式是“先登记名字，等赋值后才有函数”。
