# 056. [中级] 箭头函数中的 `this` 指向问题

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

箭头函数没有自己的 `this`。它内部的 `this` 来自定义时所在的外层词法作用域，而不是调用它的对象。

一句话回答：普通函数的 `this` 看“谁调用”，箭头函数的 `this` 看“在哪定义”。

## 问题意图

这题通常用来区分“背 this 规则”和“能判断真实代码输出”。面试官会追问对象方法、定时器、事件处理、`call/apply/bind` 是否能改变箭头函数 `this`。

## 考察范围

- 词法 `this`。
- 普通函数的动态 `this`。
- `call/apply/bind` 对箭头函数无效。
- 对象方法和回调中的差异。
- 类、定时器、事件处理里的取舍。

## 技术错误纠正

“箭头函数没有 this”这句话容易误导。准确说法是：箭头函数没有自己的 `this` 绑定，它会沿外层作用域查找 `this`。

## 知识点系统梳理

| 函数类型 | `this` 决定方式 | 能否被 `call/apply/bind` 改变 |
| --- | --- | --- |
| 普通函数 | 调用方式决定 | 可以 |
| 箭头函数 | 定义位置的外层作用域决定 | 不可以 |

判断步骤：

1. 如果是普通函数，看调用点左侧是谁。
2. 如果是箭头函数，回到它定义的位置，看外层 `this` 是什么。
3. 不要被 `obj.arrow()` 迷惑，箭头函数不会因为点调用就绑定到 `obj`。

## 实战应用举例

```js
const name = 'global'

const user = {
  name: 'Ada',
  regular() {
    return this.name
  },
  arrow: () => this.name,
}

console.log(user.regular()) // Ada
console.log(user.arrow()) // 取决于外层 this，不是 Ada
```

这个例子证明：对象方法需要当前对象作为 `this` 时，不应使用箭头函数。

```js
const counter = {
  count: 0,
  start() {
    setTimeout(() => {
      this.count += 1
      console.log(this.count)
    }, 100)
  },
}

counter.start() // 1
```

这里箭头函数适合做内部回调，因为它继承 `start()` 中的 `this`。

## 使用场景说明和对比

| 场景 | 推荐 | 原因 |
| --- | --- | --- |
| 对象公开方法 | 普通函数 | 需要调用者成为 `this` |
| 方法内部的异步回调 | 箭头函数 | 需要保留外层实例 |
| DOM 事件里需要元素本身 | 普通函数 | 浏览器会把 `this` 设为事件目标 |
| React 类属性回调 | 箭头函数可用 | 绑定实例，避免手动 bind |
| Vue `methods` | 普通函数 | Vue 会绑定组件实例 |

## 易错点提示

- `obj.arrow()` 不会让箭头函数的 `this` 指向 `obj`。
- `arrow.call(obj)` 不能改变箭头函数内部 `this`。
- 模块顶层的 `this` 和浏览器脚本顶层的 `this` 不一定一样。
- Vue `methods` 里不要用箭头函数，否则 `this` 不是组件实例。

## 记忆要点总结

- 普通函数：调用时定 `this`。
- 箭头函数：定义时借外层 `this`。
- 对象方法用普通函数。
- 回调保实例可用箭头函数。

## 延伸问题

为什么 `call` 不能改变箭头函数的 `this`？

参考答案：箭头函数创建时就没有自己的 `this` 绑定，内部 `this` 是词法捕获的外层值。`call` 只能影响有自身 `this` 绑定的普通函数。

## 可能类似的问题及简要参考答案

问：箭头函数里的 `this` 是定义时确定还是执行时确定？

答：定义时由外层词法作用域确定，不随调用方式改变。

问：定时器里为什么常用箭头函数？

答：因为定时器回调经常需要访问外层对象或组件实例，箭头函数能保留外层 `this`。

## 辅助记忆总结

记成一句话：普通函数问“谁叫我”，箭头函数问“我生在哪”。
