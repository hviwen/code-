# 073. [初级]** 模板字符串支持多行吗？

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

支持。模板字符串（反引号 `` ` ``）会原样保留源码中的所有换行和空白字符，不需要手动写 `\n`。

一句话答法：模板字符串天然支持多行，反引号之间的换行会直接变成输出中的 `\n`，但缩进空白也会一并保留，需要注意格式控制。

## 问题意图

这道题主要考察两件事：

1. 是否知道模板字符串的多行能力来自"字面换行即真换行"，而不是某种特殊语法。
2. 是否意识到缩进空白会被保留这一常见陷阱，以及 `trim()` 等清理手段。

## 考察范围

- 模板字符串中字面换行与 `\n` 的等价关系。
- 与普通字符串 `\n` 拼接的对比。
- 缩进空白被保留的副作用。
- `trim()` / `trimStart()` 清理首尾空白。
- HTML 模板、SQL 等多行文本的实际用法。
- 标签模板函数对空白的自定义处理。

## 技术错误纠正

- "模板字符串不需要转义字符"说法不够准确：模板字符串内仍然可以使用 `\n`、`\t` 等转义字符，只是多行场景下不需要靠 `\n` 来换行。
- 缩进保留不是 bug，是规范行为——反引号之间的所有字符（包括空格、制表符、换行）都是字符串内容的一部分。

## 知识点系统梳理

### 普通字符串 vs 模板字符串 vs 模板 + trim

```js
// 1. 普通字符串：必须手动 \n
const a = '第一行\n第二行\n第三行'

// 2. 模板字符串：直接换行
const b = `第一行
第二行
第三行`

// 3. 模板 + trim：去掉首尾多余换行
const c = `
第一行
第二行
第三行
`.trim()

// a === b === c  → true
```

### 缩进白会被保留

```js
function getHtml() {
  // ⚠️ 每行前面的 4 个空格会成为字符串内容
  const html = `
    <div>
      <p>hello</p>
    </div>
  `
  return html
}
// 输出包含前导空格："\n    <div>\n      <p>hello</p>\n    </div>\n  "
```

解决方式：

1. 反引号内容顶格写（牺牲代码缩进美观）。
2. 用 `.trim()` 去首尾空白。
3. 用工具函数（如 `dedent`）按最小公共缩进剥离前导空格。

### 转义字符仍然可用

```js
const mixed = `第一行\n第二行
第三行`
// "第一行\n第二行\n第三行" — \n 和字面换行效果相同
```

## 实战应用举例

### 示例 1：生成 HTML 片段

```js
function userCard(name, role) {
  return `
<div class="card">
  <h2>${name}</h2>
  <span>${role}</span>
</div>`.trimStart()
}

console.log(userCard('Alice', 'Admin'))
// 输出：
// <div class="card">
//   <h2>Alice</h2>
//   <span>Admin</span>
// </div>
```

边界：模板内容顶格写避免前导空格污染 HTML；用 `trimStart()` 去掉第一个换行。

### 示例 2：多行日志 / 错误信息

```js
function formatError(code, msg, stack) {
  return `[ERROR ${code}] ${msg}
Stack:
${stack}`.trim()
}

console.log(formatError(404, 'Not Found', '  at fetch (app.js:12)'))
// 输出：
// [ERROR 404] Not Found
// Stack:
//   at fetch (app.js:12)
```

取舍：简单场景直接用模板字符串；如果需要严格控制缩进（如生成 YAML、Python），应使用 `dedent` 类工具或数组 `.join('\n')`。

## 使用场景说明和对比

| 场景 | 推荐方式 | 原因 |
| --- | --- | --- |
| 简单多行文本 | 模板字符串 | 直观，无需 `\n` |
| HTML / SQL 模板 | 模板字符串 + `trim()` | 可读性好，去首尾空白 |
| 需要严格控制缩进的输出 | `dedent` 工具 或 数组 `.join('\n')` | 避免前导空格污染 |
| 单行拼接少量变量 | 模板字符串 | 比 `+` 拼接清晰 |
| 不含变量的纯单行字符串 | 普通引号 | 没必要用模板字符串 |

## 易错点提示

1. **缩进空白被保留**：函数体内写模板字符串，每行的前导空格会成为输出的一部分。
2. **首尾多出空行**：反引号紧接换行会在字符串开头多一个 `\n`，结尾同理。
3. **Tab 和空格混用**：不同编辑器显示不同，输出的空白宽度可能不一致。
4. **`trim()` 只去首尾**：中间行的缩进不受影响，需要 `dedent` 类方案。
5. **嵌套模板字符串的可读性**：模板内嵌 `${arr.map(x => `...`).join('')}` 多层反引号容易出错，复杂时应提取为函数。

## 记忆要点总结

- 模板字符串中的字面换行 = 输出中的 `\n`，无需手动转义。
- 反引号内的所有空白（空格、制表符、换行）都会保留。
- 用 `.trim()` 去首尾多余空白，用 `dedent` 去公共缩进。
- HTML/SQL 等多行文本是模板字符串最常见的实战场景。

## 可能类似的问题及简要参考答案

**Q：模板字符串和普通字符串拼接 `\n` 的结果一样吗？**
A：字符串内容一样。`\`第一行\n第二行\`` 和 `'第一行\n第二行'` 的值完全相等。

**Q：模板字符串中的缩进空格怎么去掉？**
A：简单场景用 `.trim()` 去首尾；需要去每行前导空格时，用 `dedent` 类工具按最小公共缩进剥离。

**Q：模板字符串可以包含反引号吗？**
A：可以，用 `\`` 转义即可。

## 辅助记忆总结

记成一句话：反引号里直接换行就是真换行，但缩进空白也会跟着进去——用 `trim()` 或 `dedent` 清理。
