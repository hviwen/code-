# 078. [初级]** ES6函数参数默认值的用法

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

ES6 默认参数的核心不是"给参数赋个初始值"，而是一套有明确触发条件、惰性求值和独立作用域的参数初始化机制。它只在参数值 `=== undefined` 时触发，`null`、`0`、`''` 都不会触发。

一句话答法：`function f(a = expr) {}` 在 `a === undefined` 时对 `expr` 惰性求值，参数列表构成独立 TDZ 作用域，后参数可引用前参数。

## 问题意图

这道题主要考察三件事：

1. 是否清楚触发条件是 `=== undefined` 而非所有 falsy 值，能否区分默认参数与 `||`/`??`。
2. 是否知道默认值表达式是惰性求值（每次调用时重新计算），以及参数间可以引用。
3. 是否了解参数列表的 TDZ 作用域，能说清参数默认值中的变量查找规则。

## 考察范围

- 基本语法 `function f(a = expr) {}`。
- 触发条件：仅 `=== undefined`，`null` 不触发。
- 惰性求值：默认值表达式在每次调用时重新计算。
- 参数列表的 TDZ 作用域：参数不能引用后面的参数或函数体内的变量。
- 后参数引用前参数：`function f(a, b = a * 2) {}`。
- 与 `||`、`??` 的对比。

## 技术错误纠正

- 默认参数不是"需要时才计算"这么模糊——准确地说是"对应参数为 `undefined` 时，在调用时对表达式求值"。每次调用都重新求值，不是只求一次。
- `null` 不会触发默认值，这是和 `??` 的关键区别。`??` 对 `null` 和 `undefined` 都走右侧，默认参数只对 `undefined`。
- 对象作为默认值时，每次调用生成新对象，不存在 Python 式的可变默认参数陷阱。

## 知识点系统梳理

### 基本语法

```js
function greet(name = 'Guest') {
  return `Hello, ${name}!`
}

greet()          // "Hello, Guest!"
greet(undefined) // "Hello, Guest!"  — undefined 触发默认值
greet(null)      // "Hello, null!"   — null 不触发
greet('')        // "Hello, !"       — 空串不触发
greet(0)         // "Hello, 0!"      — 0 不触发
```

### 触发条件：仅 `=== undefined`

```js
function test(x = 'default') { return x }

test()          // 'default'  — 未传参，x 为 undefined
test(undefined) // 'default'  — 显式传 undefined
test(null)      // null       — null ≠ undefined，不触发
test(false)     // false
test(0)         // 0
test('')        // ''
```

### 惰性求值

默认值表达式在每次函数调用时重新求值，不是定义时求一次：

```js
let id = 0
function createItem(name, itemId = ++id) {
  return { name, itemId }
}

createItem('A') // { name: 'A', itemId: 1 }
createItem('B') // { name: 'B', itemId: 2 }
createItem('C', 99) // { name: 'C', itemId: 99 } — 传了值，++id 不执行，id 仍为 2
```

### 后参数引用前参数

```js
function createRange(start = 0, end = start + 10) {
  return [start, end]
}

createRange()      // [0, 10]
createRange(5)     // [5, 15]
createRange(5, 20) // [5, 20]
```

反过来不行——前参数不能引用后参数（TDZ）：

```js
// function bad(a = b, b = 1) {} — 调用 bad() 时 ReferenceError
```

### 参数列表的 TDZ 作用域

参数默认值在一个独立的中间作用域中求值，既不在函数体作用域内，也不在外部作用域内：

```js
let x = 'outer'

function demo(x, y = x) {
  // y 拿到的是参数 x，不是外部的 'outer'
  return y
}

demo('inner') // 'inner'
```

```js
function tdz(a = b, b = 1) {}
tdz() // ReferenceError: Cannot access 'b' before initialization
```

### 默认参数 vs `||` vs `??`

| 对比项 | 默认参数 `= expr` | `\|\|` | `??` |
| --- | --- | --- | --- |
| 触发条件 | `=== undefined` | 所有 falsy 值 | `null` 或 `undefined` |
| `null` 是否触发 | 否 | 是 | 是 |
| `0`/`''`/`false` 是否触发 | 否 | 是 | 否 |
| 求值时机 | 每次调用时惰性求值 | 运行到该表达式时 | 运行到该表达式时 |
| 典型用途 | 函数签名声明意图 | ES5 兼容的参数默认 | 函数体内精确处理 null/undefined |

### 必需参数检查技巧

```js
function required(name) {
  throw new Error(`Missing parameter: ${name}`)
}

function createUser(name = required('name'), role = 'user') {
  return { name, role }
}

createUser()       // Error: Missing parameter: name
createUser('John') // { name: 'John', role: 'user' }
```

## 实战应用举例

### 示例 1：带默认配置的请求函数

这个例子展示默认参数在 API 函数中的典型用法，以及对象默认值的合并模式。

```js
function request(url, options = {}) {
  const { method = 'GET', timeout = 5000, headers = {} } = options
  console.log(`${method} ${url} timeout=${timeout}`)
  return { url, method, timeout, headers }
}

request('/api/users')
// "GET /api/users timeout=5000"

request('/api/users', { method: 'POST', timeout: 3000 })
// "POST /api/users timeout=3000"

request('/api/users', { method: 'DELETE' })
// "DELETE /api/users timeout=5000"  — timeout 走解构默认值
```

边界说明：

- `options = {}` 保证未传第二参数时解构不报错。
- 内层解构的 `method = 'GET'` 是解构默认值，和参数默认值是两层机制。
- 传 `null` 作为 `options` 会导致解构报错，因为 `null` 不触发 `= {}`。

### 示例 2：惰性求值生成唯一 ID

```js
function addTodo(text, id = crypto.randomUUID()) {
  return { id, text, done: false }
}

const a = addTodo('Buy milk')
const b = addTodo('Read book')
a.id === b.id // false — 每次调用重新生成

const c = addTodo('Fix bug', 'custom-id-001')
c.id // 'custom-id-001' — 传了值就不生成
```

边界说明：`crypto.randomUUID()` 只在参数为 `undefined` 时调用，不会浪费计算。

## 使用场景说明和对比

| 场景 | 推荐方式 | 原因 |
| --- | --- | --- |
| 函数参数有合理初始值 | 默认参数 `= expr` | 语义清晰，看签名就知道默认行为 |
| 需要区分"未传"和"传了 null" | 默认参数 | 只对 `undefined` 触发，`null` 保留 |
| 需要对 `null` 和 `undefined` 都兜底 | `??` | `??` 覆盖两者 |
| ES5 环境兼容 | `\|\|` | 但要注意 `0`/`''`/`false` 被吞 |
| 配置对象合并 | 默认参数 + 解构 | `function f(opts = {}) { const { a = 1 } = opts }` |
| 必需参数校验 | 默认参数调 throw 函数 | `f(x = required('x'))` |

## 易错点提示

- **`null` 不触发默认值**：`f(null)` 时参数值是 `null`，不是默认值。这是最常见的面试陷阱。
- **每次调用重新求值**：`function f(arr = [])` 每次调用得到新数组，不像 Python 会共享同一个列表——但反过来也意味着默认值表达式有副作用时每次都会执行。
- **TDZ：前参数不能引用后参数**：`function f(a = b, b = 1)` 调用 `f()` 会 ReferenceError。
- **`arguments` 与默认参数解耦**：使用默认参数时，`arguments` 不再与命名参数联动（严格模式下本就不联动）。`arguments[0]` 仍是 `undefined`，即使参数拿到了默认值。
- **`length` 属性受影响**：`function f(a, b = 1, c) {}` 的 `f.length` 是 `1`，只计算第一个有默认值的参数之前的参数数量。
- **解构默认值 vs 参数默认值混淆**：`function f({ a = 1 } = {})` 有两层默认值，面试中容易搞混触发条件。

## 记忆要点总结

- 触发条件是 `=== undefined`，`null` 不触发。
- 惰性求值：每次调用时重新计算默认值表达式。
- 参数列表有独立 TDZ 作用域，后参数可引用前参数，反之不行。
- `function.length` 只计数第一个默认参数之前的参数。
- 和 `||` 的区别：`||` 会吞 `0`/`''`/`false`；和 `??` 的区别：`??` 对 `null` 也走右侧。

## 可能类似的问题及简要参考答案

**Q：ES6 默认参数和 `a = a || defaultValue` 有什么区别？**  
A：默认参数只在 `a === undefined` 时触发；`||` 在所有 falsy 值（`0`、`''`、`false`、`null`）时都走右侧。传 `0` 时默认参数保留 `0`，`||` 会覆盖为 `defaultValue`。

**Q：默认参数的默认值表达式什么时候执行？**  
A：在函数被调用时、对应参数为 `undefined` 时执行。每次调用都重新求值，不是定义时求一次。

**Q：`function.length` 和默认参数的关系？**  
A：`length` 只计算第一个有默认值的参数之前的参数数量。`(a, b = 1, c).length` 是 `1`，因为 `b` 是第一个有默认值的参数。
