# 110. [中级] Map 的键可以是什么类型？

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

Map 的键可以是**任意类型**——对象、函数、NaN、undefined、Symbol 等都可以作为键。Map 使用 `SameValueZero` 算法（`===` + NaN 视为相等）判断键是否相同。这和 Object 完全不同（Object 键自动转字符串）。

一句话答法：Map 的键无类型限制。对象、函数、NaN、undefined、null、Symbol 都可作为键。键用 `SameValueZero` 比较——NaN 视作相等，对象按引用比较。

## 问题意图

1. 是否知道 Map 键不会自动转字符串（与 Object 的关键区别）。
2. 是否知道 Map 支持对象、NaN、undefined 等特殊键。
3. 是否理解 Map 的键比较算法（SameValueZero）。
4. 是否能举例说明 Map 键类型优势的实际场景。

## 考察范围

- 基本类型键：`string`、`number`、`boolean`、`undefined`、`null`、`Symbol`、`BigInt`。
- 引用类型键：`object`、`function`、`array`、`Map`、`Set` 等。
- 特殊键：`NaN`（视为相等）、`undefined`、`Symbol('a') !== Symbol('a')`。
- 引用比较：`{}` 和 `{}` 是两个不同键。
- 与 Object 对比：Object 键自动转字符串（`true` → `'true'`，`{}` → `'[object Object]'`）。

## 技术错误纠正

- "Map 键可以是任何类型包括 NaN"——对，但注意 NaN 比较特殊：`map.set(NaN, 1); map.get(NaN) → 1`，而 `NaN !== NaN` 在 JS 中通常成立。
- "Map 的键比 Object 好"——看场景。Map 键类型自由但 Object 有字面量语法和 JSON 支持。

## 知识点系统梳理

### 各种键类型示例

```js
const map = new Map()

// 基本类型
map.set('str', 1)        // 字符串
map.set(42, 2)           // 数字
map.set(true, 3)         // 布尔
map.set(undefined, 4)    // undefined
map.set(null, 5)         // null
map.set(Symbol('s'), 6)  // Symbol
map.set(1n, 7)           // BigInt

// 引用类型
const obj = { x: 1 }
map.set(obj, 8)          // 对象
const fn = () => {}
map.set(fn, 9)           // 函数
map.set([1, 2], 10)      // 数组

// 特殊：NaN
map.set(NaN, 'nan')
map.get(NaN)             // 'nan'（SameValueZero 把 NaN 视为相等）
```

### Map 键 vs Object 键

```js
// Object 键自动转字符串
const o = {}
o[true] = 1         // 键变为 'true'
o[{}] = 2           // 键变为 '[object Object]'
Object.keys(o)      // ['true', '[object Object]']

// Map 键保留原始类型
const m = new Map()
m.set(true, 1)      // 键是布尔 true
m.set({}, 2)        // 键是对象引用
m.get(true)         // 1
```

### 引用比较

```js
const m = new Map()
const a = { id: 1 }
const b = { id: 1 }

m.set(a, 'object a')
m.get(b)  // undefined（不同引用）
m.get(a)  // 'object a'
```

## 实战应用举例

### 示例：DOM 节点作键

```js
const state = new Map()

function initButton(btn) {
  if (!state.has(btn)) {
    state.set(btn, { clicks: 0 })
  }
}
// 只有 Map（或 WeakMap）能直接以 DOM 节点为键
// Object 会把 DOM 节点转成 '[object HTMLButtonElement]'
```

## 使用场景说明和对比

| 键类型 | Map | Object |
| --- | --- | --- |
| 字符串 | ✅ | ✅ |
| 数字 | ✅（不转换） | ✅（转为字符串） |
| 布尔值 | ✅（不转换） | ✅（转为字符串） |
| 对象 | ✅（按引用） | ❌（转为 '[object Object]'） |
| 函数 | ✅（按引用） | ❌（转为字符串） |
| NaN | ✅（视为相等） | ❌ 不支持 |
| undefined | ✅ | ✅（转为 'undefined'） |
| null | ✅ | ✅（转为 'null'） |

## 易错点提示

- Map 对象键按**引用**比较：`{a:1}` 和 `{a:1}` 是两个不同的键。
- NaN 在 Map 中作为键时被视为相等：`map.set(NaN, 'x'); map.get(NaN) → 'x'`。
- `undefined` 可以作为 Map 键（与 Object 不同，Object 中 `obj[undefined]` 也会变成 `obj['undefined']`）。
- Symbol 键在 Map 中是唯一的：`Symbol('a') !== Symbol('a')`。

## 记忆要点总结

- Map 键：任意类型，不自动转换，NaN 视为相等，对象按引用。
- Object 键：只限字符串/Symbol，自动转字符串。
- Map 优势：能用对象/函数/NaN 作键。

## 可能类似的问题及简要参考答案

**Q：Map 的键能是对象吗？**
A：能。对象键按引用比较，两个不同对象即使内容相同也是不同键。

**Q：为什么 Object 不能用对象作键？**
A：Object 的键会通过 `toString()` 转为字符串，`{}` 变成 `'[object Object]'`。

## 辅助记忆总结

记成一句话：Map 的键不挑食——任何类型都行，Object 只吃字符串和 Symbol。
