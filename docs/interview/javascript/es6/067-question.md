# 067. [中级]** 如何交换两个变量的值（使用解构）？

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

解构交换的核心不是"少写一行代码"，而是用 `[a, b] = [b, a]` 一条语句同时完成取值和赋值，消除临时变量，语义上直接表达"交换"。

一句话答法：`[a, b] = [b, a]` 利用数组解构，在右侧先构造包含旧值的临时数组，再同时赋给左侧变量，实现原子交换。

## 问题意图

这道题主要考察两件事：

1. 是否理解解构赋值的求值顺序——右侧表达式先全部求值，再逐一赋给左侧。
2. 是否知道 `let`/`var` 和 `const` 在交换场景下的限制，以及多变量、数组元素交换等扩展写法。

## 考察范围

- `[a, b] = [b, a]` 基本语法和求值顺序。
- 多变量循环交换：`[a, b, c] = [c, a, b]`。
- 数组元素交换：`[arr[i], arr[j]] = [arr[j], arr[i]]`。
- 与临时变量法、XOR 法、算术法的对比。
- `let`/`var` 才能交换，`const` 不行。
- 行首分号陷阱（ASI 问题）。

## 技术错误纠正

- 代码示例中应该使用 `let` 而不是 `const`，因为交换需要重新赋值，`const` 会抛 `TypeError`。
- 正确语法是 `[x, y] = [y, x]`，不要漏掉赋值号。
- 如果上一行没有分号，`[a, b] = [b, a]` 可能被解析为属性访问。独立使用时行首加 `;` 是必要的：`;[a, b] = [b, a]`。

## 知识点系统梳理

### 基本语法

```js
let a = 1
let b = 2

;[a, b] = [b, a]

console.log(a, b) // 2 1
```

求值顺序：右侧 `[b, a]` 先求值生成临时数组 `[2, 1]`，然后解构赋值给 `a` 和 `b`。

### 多变量交换

```js
let x = 1, y = 2, z = 3

;[x, y, z] = [z, x, y]

console.log(x, y, z) // 3 1 2
```

等价于三个变量的循环轮换，一条语句完成。

### 数组元素交换

```js
const arr = [10, 20, 30]

;[arr[0], arr[2]] = [arr[2], arr[0]]

console.log(arr) // [30, 20, 10]
```

注意：数组本身用 `const` 声明没问题，交换的是元素而非变量绑定。

### 方法对比

| 方法 | 代码 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 临时变量 | `let t = a; a = b; b = t` | 最直观，无性能顾虑 | 多一个变量，多两行 |
| 解构赋值 | `[a, b] = [b, a]` | 一行搞定，语义清晰 | 右侧创建临时数组，极端热路径有微小开销 |
| XOR 位运算 | `a ^= b; b ^= a; a ^= b` | 无额外内存 | 仅限整数，可读性差，自身交换出 bug |
| 算术加减 | `a = a + b; b = a - b; a = a - b` | 无额外内存 | 有溢出风险，仅限数值 |

日常开发推荐解构赋值；排序算法热循环里如果 benchmark 证明有差异再换临时变量。

## 实战应用举例

### 示例 1：冒泡排序中的元素交换

```js
function bubbleSort(arr) {
  const result = [...arr]
  for (let i = 0; i < result.length - 1; i++) {
    for (let j = 0; j < result.length - 1 - i; j++) {
      if (result[j] > result[j + 1]) {
        ;[result[j], result[j + 1]] = [result[j + 1], result[j]]
      }
    }
  }
  return result
}

console.log(bubbleSort([64, 25, 12, 22, 11]))
// [11, 12, 22, 25, 64]
```

边界说明：

- 交换发生在数组元素上，`result` 用 `const` 声明即可。
- 如果数组量级到百万且排序是瓶颈，benchmark 后可改回临时变量——V8 通常已优化掉临时数组分配，差异极小。

### 示例 2：Fisher-Yates 洗牌

```js
function shuffle(arr) {
  const result = [...arr]
  for (let i = result.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1))
    ;[result[i], result[j]] = [result[j], result[i]]
  }
  return result
}

console.log(shuffle([1, 2, 3, 4, 5]))
// 随机顺序，如 [3, 1, 5, 2, 4]
```

边界说明：

- `i === j` 时交换自身，解构赋值正常工作（XOR 法此时会归零出 bug）。
- 对比：这里解构的可读性优势明显，一眼就能看出"交换两个位置"。

## 使用场景说明和对比

| 场景 | 是否适合解构交换 | 原因 |
| --- | --- | --- |
| 两个变量交换 | 适合 | 最简洁，一行完成 |
| 排序/洗牌中的元素交换 | 适合 | 可读性好，性能差异可忽略 |
| 多变量循环轮换 | 适合 | 临时变量法需要多个中间量 |
| 仅限整数的极端性能场景 | 不适合 | 临时变量或 XOR 更快（需 benchmark 证明） |
| `const` 声明的变量 | 不可用 | `const` 不允许重新赋值 |
| 对象属性交换 | 可用 | `[obj.a, obj.b] = [obj.b, obj.a]` 合法 |

## 易错点提示

- 用 `const` 声明的变量无法交换，必须用 `let` 或 `var`。
- 行首 `[` 可能被解析为上一行的属性访问，独立语句时前面加 `;`。
- 解构交换右侧每次会创建一个临时数组，正常场景可忽略，但了解机制有助于回答追问。
- XOR 交换相同变量引用（`a ^= a`）会得到 0，解构交换不会有这个问题。
- 多变量交换时左右两侧的顺序决定了轮换方向，写反了结果就反了。
- 不要在解构交换中混入副作用表达式（如函数调用），求值顺序虽然确定但可读性极差。

## 记忆要点总结

- `[a, b] = [b, a]`——右侧先全部求值，再赋左侧。
- 变量必须是 `let`/`var`，`const` 不行。
- 数组元素交换用 `[arr[i], arr[j]] = [arr[j], arr[i]]`，数组本身可以是 `const`。
- 日常开发用解构，极端性能用临时变量。
- 行首加 `;` 防 ASI 歧义。

## 可能类似的问题及简要参考答案

**Q：不用第三个变量，如何交换两个数的值？**  
A：`[a, b] = [b, a]`。如果要求不用解构，可以用 `a = a + b; b = a - b; a = a - b`（仅限数值且有溢出风险），或 XOR（仅限整数）。推荐解构。

**Q：`const` 声明的变量能用解构交换吗？**  
A：不能。`const` 禁止重新赋值，`[a, b] = [b, a]` 会抛 `TypeError: Assignment to constant variable`。

**Q：解构赋值和临时变量交换哪个性能更好？**  
A：现代引擎差异极小。V8 等引擎可能将临时数组优化为寄存器操作。日常选可读性更好的解构；只有 benchmark 证明瓶颈时才考虑临时变量。

## 辅助记忆总结

记成一句话：`[a, b] = [b, a]` 就是"右边先算完，左边再接住"，一行完成交换，记得用 `let` 和行首分号。
