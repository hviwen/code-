# 105. [中级] 如何用 Set 实现数组去重？

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

利用 Set 的**值唯一性**特性，将数组传入 `new Set()` 自动去重，再转回数组。这是 Set 最经典的应用场景。面试问这个是在考察是否理解 Set 的去重规则（`SameValueZero` 算法：`===` + NaN 特殊处理）。

一句话答法：`[...new Set(arr)]` 或 `Array.from(new Set(arr))`。利用 Set 自动去重特性，将数组作为可迭代对象传入构造函数。

## 问题意图

1. 是否知道 Set 构造函数接受可迭代对象（数组天然可迭代）。
2. 是否知道扩展运算符 `...` 对可迭代对象的展开能力。
3. 是否理解 Set 的去重是基于 `SameValueZero` 算法（NaN 去重、对象按引用）。

## 考察范围

- `new Set(arr)` 创建 Set 时自动去重。
- `[...set]` 或 `Array.from(set)` 转回数组。
- 去重规则：NaN 去重（`new Set([NaN, NaN]).size === 1`）。
- 对象按引用去重：`{a:1}` 和 `{a:1}` 视为不同元素。
- 字符串去重 vs 数字去重：`'1'` 和 `1` 在 Set 中被视为不同值（类型不同）。
- 不能去重深层嵌套的对象（按引用比较）。

## 技术错误纠正

- "Set 能去重所有类型"——不是。对象按引用比较，两个看起来一样的对象仍是两个独立元素。
- 原始代码 `[...new Set(arr)]` 是正确的去重方式。

## 知识点系统梳理

### 去重规则详解

```js
// SameValueZero 算法
// 规则：=== 全等 + NaN 视为相等

// 1. 基本类型
;[...new Set([1, '1', 1, '1'])]   // [1, '1'] — 类型不同不重复
;[...new Set([NaN, NaN])]          // [NaN] — NaN 视为相等
;[...new Set([null, undefined])]   // [null, undefined] — 不同值

// 2. 引用类型
const a = { id: 1 }
const b = { id: 1 }
const c = a
;[...new Set([a, b, c])]           // [{id:1}, {id:1}] — a 和 b 不同引用
```

### 去重方法对比

```js
const arr = [1, 2, 2, 3, 3, 3]

// ✅ Set（推荐）
;[...new Set(arr)]          // [1, 2, 3] — 简洁 O(n)

// filter + indexOf（早期替代）
arr.filter((v, i) => arr.indexOf(v) === i) // [1, 2, 3] — O(n²)

// reduce + 缓存
arr.reduce((acc, v) => acc.includes(v) ? acc : [...acc, v], []) // O(n)

// Object 键缓存
Object.keys(arr.reduce((acc, v) => (acc[v] = 1, acc), {})) // 丢失类型
```

### 对象数组去重

```js
// 按某个字段去重
const items = [
  { id: 1, name: 'a' },
  { id: 2, name: 'b' },
  { id: 1, name: 'a copy' },
]

// Map 按 id 去重
const unique = [...new Map(items.map(item => [item.id, item])).values()]
// [{id:1, name:'a'}, {id:2, name:'b'}]

// 或者同一引用
const seen = new Set()
const result = items.filter(item => {
  if (seen.has(item.id)) return false
  seen.add(item.id)
  return true
})
```

## 实战应用举例

### 示例 1：基本去重

```js
const arr = [1, 2, 2, 3, 3, 3]
const unique = [...new Set(arr)]
// [1, 2, 3]
```

### 示例 2：复杂场景——合并去重

```js
const tags1 = ['js', 'react', 'vue']
const tags2 = ['react', 'angular', 'vue', 'svelte']
const merged = [...new Set([...tags1, ...tags2])]
// ['js', 'react', 'vue', 'angular', 'svelte']
```

## 使用场景说明和对比

| 场景 | 方案 | 说明 |
| --- | --- | --- |
| 基本类型数组去重 | `[...new Set(arr)]` | 一行搞定 |
| 对象数组按字段去重 | `Map` 或 `filter + Set.has` | 无法用 Set 直接去重 |
| 需要去重同时保持类型 | `[...new Set(arr)]` | 类型不同不合并 |
| 大量数据去重 | `Set` | O(n)，比 filter O(n²) 快得多 |

## 易错点提示

- `[1, '1']` 在 Set 中是两个元素——类型不同不视为重复。
- 对象去重只按引用：`{id:1} !== {id:1}`，Set 不会去重。
- `[{a: 1}, {a: 1}]` 用 Set 去重无效，需要用 `JSON.stringify` 或特定字段。
- NaN 在 Set 中视为相等：`new Set([NaN, NaN]).size === 1`，这和 `NaN !== NaN` 不同。

## 记忆要点总结

- `[...new Set(arr)]` 一行去重。
- 规则：基本类型按值 ===，NaN 特殊处理（视为相等），对象按引用。
- 对象去重需要 Map 按字段或 JSON.stringify。

## 可能类似的问题及简要参考答案

**Q：Set 去重能去重对象吗？**
A：不能。对象按引用比较，`{a:1}` 和 `{a:1}` 是两个不同对象。

**Q：`[NaN, NaN]` 用 Set 去重会保留几个？**
A：一个。Set 使用 SameValueZero 算法，将 NaN 视为相等。

**Q：Set 和 filter 去重哪个更快？**
A：Set 是 O(n)，filter+indexOf 是 O(n²)。数据量大时 Set 快很多。

## 辅助记忆总结

记成一句话：`[...new Set(arr)]` 是最简洁的去重方案——基本类型按值去重，对象按引用，NaN 特殊视作相等。
