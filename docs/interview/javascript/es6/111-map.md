# 111. [中级] 如何将 Map 转换为数组？

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

Map 实现了可迭代协议，所以 `[...map]` 即可将其转换为 `[key, value]` 对的数组。也可以用 `Array.from(map)`。根据需求还可以提取键数组、值数组或转为对象。

一句话答法：`[...map]` 得到 `[[key, val], ...]` 数组，`[...map.keys()]` 得键数组，`[...map.values()]` 得值数组。转对象用 `Object.fromEntries(map)`。

## 问题意图

1. 是否知道扩展运算符和 `Array.from` 对 Map 的作用。
2. 是否知道 `map.keys()`、`map.values()`、`map.entries()` 返回迭代器。
3. 是否知道 `Object.fromEntries(map)` 可以将 Map 转对象。
4. 是否能区分不同转换方式的结果结构。

## 考察范围

- `[...map]` — 展开 Map 得到 `[key, val]` 数组。
- `Array.from(map)` — 等价于 `[...map]`。
- `[...map.keys()]` — 键数组。
- `[...map.values()]` — 值数组。
- `[...map.entries()]` — 同 `[...map]`。
- `Object.fromEntries(map)` — Map 转对象（ES2019）。
- Map 转回数组后可以用数组方法：map、filter、reduce 等。

## 技术错误纠正

- "Map 可以直接转数组"——是的，因为 Map 实现了 `[Symbol.iterator]`。
- "`[...map]` 和 `Array.from(map)` 是一样的"——等价，但 `Array.from` 还支持映射函数作为第二个参数。

## 知识点系统梳理

### 五种转换方式

```js
const map = new Map([
  ['a', 1],
  ['b', 2],
  ['c', 3],
])

// 1. 展开为 [key, value] 数组
;[...map]             // [['a',1], ['b',2], ['c',3]]

// 2. Array.from
Array.from(map)       // [['a',1], ['b',2], ['c',3]]

// 3. 仅键
;[...map.keys()]      // ['a', 'b', 'c']

// 4. 仅值
;[...map.values()]    // [1, 2, 3]

// 5. 转对象
Object.fromEntries(map) // { a: 1, b: 2, c: 3 }
```

### 链式处理

```js
const map = new Map([['a', 1], ['b', 2], ['c', 3]])

// 转数组后可以链式操作
;[...map]
  .filter(([k, v]) => v > 1)
  .map(([k, v]) => ({ key: k, value: v }))
// [{ key: 'b', value: 2 }, { key: 'c', value: 3 }]

// Array.from 支持映射
Array.from(map, ([k, v]) => `${k}=${v}`)
// ['a=1', 'b=2', 'c=3']
```

### Map ↔ 对象 ↔ 数组

```js
// 对象 → Map
new Map(Object.entries({ a: 1, b: 2 }))  // Map {'a' => 1, 'b' => 2}

// Map → 对象
Object.fromEntries(map)  // { a: 1, b: 2 }

// Map → 数组
;[...map]  // [['a',1], ['b',2]]
```

## 实战应用举例

### 示例：Map 过滤并排序

```js
const scores = new Map([
  ['alice', 85],
  ['bob', 92],
  ['charlie', 78],
])

// 过滤出 80 分以上，按分数降序
const result = [...scores]
  .filter(([name, score]) => score >= 80)
  .sort(([, a], [, b]) => b - a)
// [['bob', 92], ['alice', 85]]
```

## 使用场景说明和对比

| 需求 | 写法 |
| --- | --- |
| 完整的 [key, value] 数组 | `[...map]` |
| 仅提取键列表 | `[...map.keys()]` |
| 仅提取值列表 | `[...map.values()]` |
| 转换为对象 | `Object.fromEntries(map)` |
| 转换时做映射 | `Array.from(map, fn)` |

## 易错点提示

- `Object.fromEntries` 只适用于键为字符串的 Map，Symbol 键会被忽略。
- `[...map]` 的是 `[key, value]` 对组成的数组，不是扁平的值列表。
- `Array.from(map)` 第二个参数是映射函数，不是 thisArg。
- Map 转对象时如果有同名键后面的会覆盖前面的（但 Map 键唯一，基本不会发生）。

## 记忆要点总结

- `[...map]` → `[[k,v], ...]` 数组。
- `[...map.keys()]` → 键数组，`[...map.values()]` → 值数组。
- `Object.fromEntries(map)` → 对象。
- `new Map(Object.entries(obj))` → 对象转 Map。

A：可以，`Object.fromEntries(map)`，但键必须是字符串或 Symbol。
