# 104. [中级] Set 的基本用法和特性

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

Set 是 ES6 新增的**值集合**数据结构，元素唯一（严格相等 `===` 去重，NaN 视为相等）。核心价值是自动去重和 O(1) 查找。区别在于 Set 按插入顺序遍历且元素唯一，数组允许重复且需 `indexOf/includes` 手动去重。

一句话答法：Set 是元素唯一的有序集合，`new Set(iterable)` 创建，`add/delete/has/size` 操作，NaN 视作相等，按插入顺序遍历。

## 问题意图

1. 是否知道 Set 的**值唯一**特性及去重算法（基于 `SameValueZero`，NaN === NaN）。
2. 是否熟悉 Set 的增删查 API：`add`、`delete`、`has`、`clear`、`size`。
3. 是否知道 Set 可遍历：`forEach`、`keys`、`values`、`entries`（与 Map 保持对称 API）。

## 考察范围

- `new Set()` / `new Set(iterable)`。
- `set.add(val)`、`set.delete(val)`、`set.has(val)`、`set.clear()`、`set.size`。
- Set 的值唯一性：NaN 视为相等（`NaN === NaN` 在 Set 中为 `true`），引用类型按引用比较。
- 按插入顺序遍历：`set.forEach`、`for...of`、`set.keys()`、`set.values()`、`set.entries()`。
- `keys()` 和 `values()` 返回相同迭代器（为与 Map 保持对称）。
- Set 转数组：`[...set]`、`Array.from(set)`。
- WeakSet（仅对象，弱引用）与 Set 的区别。

## 技术错误纠正

- 原始代码中 `set[0]` 做遍历——Set 没有索引，不能用 `set[0]` 或方括号访问。
- "Set 像数组但不重复"——Set 是集合不是索引序列，没有索引访问。

## 知识点系统梳理

### 基本操作

```js
// 创建
const s = new Set([1, 2, 3, 1, 2])  // Set {1, 2, 3}
new Set('hello')                     // Set {'h', 'e', 'l', 'o'}

// 增删查
s.add(4)        // Set {1, 2, 3, 4}
s.has(2)        // true
s.delete(2)     // true（删除成功返回 true）
s.size          // 3
s.clear()       // 清空
```

### 唯一性规则

```js
// 严格等于（===）+ NaN 特殊处理
const s = new Set()
s.add(NaN)
s.add(NaN)      // 只保留一个 NaN（Set 认为 NaN 相等）
s.size          // 1

s.add({ a: 1 })
s.add({ a: 1 }) // 不同引用 → 可以同时存在
s.size          // 3（NaN + obj1 + obj2）
```

### 遍历

```js
const s = new Set(['a', 'b', 'c'])

s.forEach(v => console.log(v))    // a, b, c
for (const v of s)                // a, b, c
s.keys()                          // 同 values↓
s.values()                        // 'a', 'b', 'c' 的迭代器
s.entries()                       // ['a','a'], ['b','b'], ['c','c']（与 Map 对称）

// 转数组
;[...s]            // ['a', 'b', 'c']
Array.from(s)      // ['a', 'b', 'c']
```

### Set vs Array

| 对比 | Set | Array |
| --- | --- | --- |
| 元素唯一性 | 自动去重 | 不保证 |
| 索引访问 | 不⽀持 | `arr[i]` |
| 查找效率 | `has()` O(1) | `includes()` O(n) |
| 删除效率 | `delete()` O(1) | `splice/filter` O(n) |
| 遍历顺序 | 插入顺序 | 索引顺序 |
| NaN 处理 | 视为相等 | `includes` 也视为相等 |

## 实战应用举例

### 示例 1：数组去重

```js
const nums = [1, 2, 2, 3, 3, 3]
;[...new Set(nums)]  // [1, 2, 3]
```

### 示例 2：交集/差集

```js
const a = new Set([1, 2, 3])
const b = new Set([2, 3, 4])

// 交集
new Set([...a].filter(x => b.has(x)))  // Set {2, 3}
// 差集
new Set([...a].filter(x => !b.has(x))) // Set {1}
```

## 使用场景说明和对比

| 场景 | 推荐 | 原因 |
| --- | --- | --- |
| 数组去重 | `new Set()` | 一行搞定 |
| 快速存在性检查 | `Set.has()` | O(1) 查找 |
| 维护不重复集合 | `Set` | 自动去重 |
| 需要索引访问 | `Array` | Set 不⽀持 |
| 需要保持元素顺序的集合 | `Set` | 按插入顺序遍历 |

## 易错点提示

- Set 没有索引下标——不能用 `set[0]`。要获取值需遍历（`for...of` 或 `forEach`）。
- Set 的删除返回布尔值：`true`（存在并删除）/ `false`（不存在）。
- Set 的 NaN 去重：`new Set([NaN, NaN])` 的 size 为 1。
- 对象按引用比较：`set.add({a:1}); set.add({a:1})` 是两个不同元素。
- Set 的 `keys()` 和 `values()` 返回相同的迭代器（为与 Map 保持 API 对称）。

## 记忆要点总结

- Set = 唯一值集合，O(1) 查/增/删。
- 创建：`new Set(iterable)`。
- 操作：`add/has/delete/clear/size`。
- 遍历：`for...of`、`forEach`、`...set`。
- NaN 视为相等，对象按引用比较。

## 可能类似的问题及简要参考答案

**Q：Set 怎么去重数组？**
A：`[...new Set(arr)]` 或 `Array.from(new Set(arr))`。

**Q：Set 能直接用索引取元素吗？**
A：不能。Set 不是索引序列，只能通过遍历获取元素。

**Q：Set 的 forEach 回调参数为什么是三个？**
A：为了与 Map 保持对称，Set 的 `forEach((value, key, set) => ...)` 中 `key === value`。

## 辅助记忆总结

记成一句话：Set 是"不重复集合"——O(1) 查删，`[...new Set(arr)]` 一行去重，没有索引只能遍历。
