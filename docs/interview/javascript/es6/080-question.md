# 080. [中级]** 如何使用扩展运算符合并数组？

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

`[...arr1, ...arr2]` 把两个（或多个）数组的元素展开到一个新数组里，结果是浅拷贝。它比 `concat` 更直观，比 `push.apply` 更安全（不修改原数组）。

一句话答法：扩展运算符通过迭代协议把可迭代对象的元素逐个放入新数组，`[...a, ...b]` 等价于 `a.concat(b)` 但语法更灵活——可以在任意位置插入元素。

## 问题意图

1. 是否掌握 `...` 在数组字面量中的展开语义，而不是和 rest 参数混淆。
2. 是否知道合并结果是浅拷贝，嵌套对象仍然共享引用。
3. 是否能对比 `concat`、`push.apply` 并说出各自适用场景。

## 考察范围

- 扩展运算符的数组展开语法。
- `[...a, ...b]` 合并多个数组。
- 浅拷贝特性（只复制第一层）。
- spread vs `concat` vs `push.apply` 的区别。
- 结合 `Set` 做去重合并。
- 在任意位置插入元素。

## 技术错误纠正

- `[...arr1, ...arr2]` 不是深拷贝，嵌套引用类型仍共享同一个对象。
- `push.apply(arr1, arr2)` 会修改 `arr1`，并且当 `arr2` 长度超过引擎参数上限（通常 ~65536）时会抛 RangeError。
- 扩展运算符只能展开可迭代对象（实现了 `Symbol.iterator`），普通对象不能在数组字面量中展开。

## 知识点系统梳理

### 基本合并

```js
const a = [1, 2, 3]
const b = [4, 5, 6]
const merged = [...a, ...b]        // [1, 2, 3, 4, 5, 6]
const multi = [...a, ...b, ...[7]] // [1, 2, 3, 4, 5, 6, 7]
```

### 在任意位置插入

```js
const head = [0, ...a]           // [0, 1, 2, 3]
const mid  = [...a, 'x', ...b]  // [1, 2, 3, 'x', 4, 5, 6]
```

### 结合 Set 去重

```js
const unique = [...new Set([...a, ...b])]
// a=[1,2,3], b=[2,3,4] => [1, 2, 3, 4]
```

注意：`Set` 用 `===` 判断（`NaN` 除外），对引用类型按引用去重，不会深比较。

### 浅拷贝本质

```js
const src = [{ id: 1 }, { id: 2 }]
const copy = [...src]
copy[0].id = 99
console.log(src[0].id) // 99 —— 共享引用
```

### 方法对比

| 对比项 | `[...a, ...b]` | `a.concat(b)` | `a.push(...b)` |
| --- | --- | --- | --- |
| 是否修改原数组 | 否，新数组 | 否，新数组 | 是，修改 `a` |
| 语法灵活性 | 可插入元素、条件展开 | 只能首尾拼接 | 只追加到末尾 |
| 大数组安全性 | 安全 | 安全 | `b` 过大可能超参数上限 |
| 可读性 | 高 | 中 | 低 |
| 兼容性 | ES6+ | ES3+ | ES6+（spread 形式） |

## 实战应用举例

### 示例 1：条件合并与默认值

```js
function buildFilters(base, extra, includeDefaults) {
  const defaults = includeDefaults ? ['all'] : []
  return [...defaults, ...base, ...extra]
}

buildFilters(['name'], ['age'], true)   // ['all', 'name', 'age']
buildFilters(['name'], ['age'], false)  // ['name', 'age']
```

边界说明：

- 条件数组用三元返回 `[]` 而不是 `undefined`，否则展开 `undefined` 会抛 TypeError。
- 如果参数可能不是数组，应在入口校验或用 `Array.isArray()`。

### 示例 2：不可变状态更新（Redux / Pinia 常见模式）

```js
// 在列表中间插入一条记录
function insertAt(list, index, item) {
  return [...list.slice(0, index), item, ...list.slice(index)]
}

const todos = ['a', 'b', 'c']
insertAt(todos, 1, 'x') // ['a', 'x', 'b', 'c']
todos                    // ['a', 'b', 'c'] —— 原数组不变
```

边界说明：

- `slice` + spread 产生新数组，适合不可变更新；大列表频繁操作可考虑 `splice` 或 Immer。

## 使用场景说明和对比

| 场景 | 推荐方式 | 原因 |
| --- | --- | --- |
| 合并两三个短数组 | `[...a, ...b]` | 简洁、不可变 |
| 需要在中间插入元素 | `[...a, x, ...b]` | `concat` 做不到 |
| 条件性合并某段数据 | `[...a, ...(cond ? b : [])]` | 三元 + spread 最清晰 |
| 超大数组（10w+）拼接 | `a.concat(b)` 或手动 `push` | spread 会展开全部元素到参数，可能更慢 |
| 合并后追加到已有数组 | `target.push(...items)` | 直接修改原数组，省一次分配 |
| 合并 + 去重 | `[...new Set([...a, ...b])]` | 一行解决基本类型去重 |

## 易错点提示

- **浅拷贝陷阱**：`[...arr]` 只复制第一层，嵌套对象仍共享引用，修改会互相影响。
- **只能展开可迭代对象**：`[...plainObj]` 会抛 TypeError，普通对象要用 `Object.values()` 或 `Object.entries()` 转换后再展开。
- **大数组性能**：`func(...hugeArray)` 会把数组元素全部作为函数参数传入，超过引擎上限（通常 ~65536）会抛 RangeError；合并大数组优先用 `concat`。
- **`push(...arr)` 和 `push(arr)` 不同**：前者逐个追加元素，后者把整个数组作为一个元素追加。
- **条件展开别忘兜底空数组**：`...(condition ? arr : [])` 中如果 falsy 分支返回 `undefined` / `null` 而不是 `[]`，会抛错。
- **Set 去重只对基本类型有效**：`[...new Set([{a:1},{a:1}])]` 不会去重，因为两个对象是不同引用。

## 记忆要点总结

- `[...a, ...b]` 创建新数组，不修改原数组。
- 结果是浅拷贝，嵌套引用共享。
- 可在任意位置插入元素，比 `concat` 更灵活。
- 大数组（10w+）用 `concat` 更安全。
- 去重用 `[...new Set([...a, ...b])]`，仅对基本类型有效。

## 可能类似的问题及简要参考答案

**Q：扩展运算符和 `concat` 有什么区别？**  
A：功能相似，都返回新数组。spread 可以在任意位置插入元素、做条件展开；`concat` 兼容 ES3，大数组性能更稳定。

**Q：如何用扩展运算符实现数组去重？**  
A：`[...new Set(arr)]`，只对基本类型有效；引用类型需要自定义比较逻辑。

**Q：扩展运算符是深拷贝吗？**  
A：不是。只复制第一层元素，嵌套的对象/数组仍然共享引用。需要深拷贝用 `structuredClone()` 或递归复制。

## 辅助记忆总结

`[...a, ...b]` 是"把两个数组的元素摊开放进一个新数组"，记住三个关键词：新数组、浅拷贝、任意位置插入。
