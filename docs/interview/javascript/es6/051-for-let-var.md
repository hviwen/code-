# 051. [中级] for 循环中使用 `let` 和 `var` 的区别

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

这道题考察的是循环变量、闭包和异步回调之间的关系。核心区别是：`var` 的循环变量属于同一个函数作用域绑定，`let` 在 `for` 循环中会为每一轮迭代创建新的词法绑定。

一句话回答：`var` 循环里的回调共享同一个 `i`，所以异步执行时常拿到最终值；`let` 每轮循环都有自己的 `i`，回调能记住当轮值。

## 问题意图

面试官通常会用 `setTimeout`、事件绑定或 Promise 回调追问，判断你是否真的理解闭包捕获的是变量绑定，而不是“当时的数值快照”。

## 考察范围

- `var` 的函数作用域。
- `let` 的块级作用域和每轮迭代绑定。
- 闭包引用变量绑定的行为。
- 异步回调执行时变量值为什么已经变化。

## 技术错误纠正

不能只说“`let` 有块级作用域，所以输出 0、1、2”。更准确的说法是：在 `for (let i = 0; ...)` 中，规范会为每一次迭代创建新的 `i` 绑定，闭包引用的是当轮绑定。

也不能说 `var` “没有作用域”。`var` 有作用域，只是它是函数作用域，不是块级作用域。

## 知识点系统梳理

| 对比点 | `var` | `let` |
| --- | --- | --- |
| 作用域 | 函数作用域 | 块级作用域 |
| 循环绑定 | 所有迭代共享一个变量 | 每轮迭代创建新绑定 |
| 异步回调中读取 | 常读到循环结束后的值 | 读到对应轮次的值 |
| 旧式解决方案 | IIFE、函数参数保存值 | 通常直接使用 |

这类题的理解路径是：

1. 回调函数形成闭包。
2. 闭包捕获变量绑定。
3. `var` 只有一个共享绑定。
4. `let` 在 `for` 迭代中有多个独立绑定。

## 实战应用举例

```js
for (var i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(i)
  }, 0)
}
// 输出：3 3 3
```

`setTimeout` 的回调会在同步循环结束后执行。循环结束时，共享的 `i` 已经变成 `3`，所以三个回调都读到同一个最终值。

```js
for (let i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(i)
  }, 0)
}
// 输出：0 1 2
```

这里每轮迭代都有自己的 `i`，三个回调引用的是三个不同绑定。

旧代码里如果必须用 `var`，可以用函数参数制造新绑定：

```js
for (var i = 0; i < 3; i++) {
  ;((index) => {
    setTimeout(() => {
      console.log(index)
    }, 0)
  })(i)
}
// 输出：0 1 2
```

## 使用场景说明和对比

真实项目中，这个问题常出现在批量绑定事件、批量发起异步任务、循环生成回调函数时。

```js
const buttons = document.querySelectorAll('button')

for (let i = 0; i < buttons.length; i++) {
  buttons[i].addEventListener('click', () => {
    console.log(`点击第 ${i + 1} 个按钮`)
  })
}
```

如果这里用 `var`，所有点击回调都可能读到循环结束后的同一个索引，导致定位错误。现代代码中直接使用 `let` 是最简单、最清晰的做法。

| 场景 | `var` 风险 | `let` 优势 |
| --- | --- | --- |
| `setTimeout` 循环打印 | 共享最终索引 | 每轮保留当轮索引 |
| 批量绑定事件 | 点击时读到同一个变量 | 每个监听器对应独立绑定 |
| Promise 回调中读索引 | 回调执行时循环已结束 | 回调读取当轮绑定 |
| 纯同步循环求和 | 通常不暴露差异 | 仍能限制变量作用域 |

## 易错点提示

`let` 不是因为“把值复制了一份”才正确，而是因为每次迭代创建了新的词法环境。

如果循环体里没有闭包或异步回调，`var` 和 `let` 的输出差异不一定明显，但仍建议用 `let` 限制作用域。

`forEach`、`map` 等数组方法的回调参数天然是每次调用的新参数，也能避免共享循环变量问题。

## 记忆要点总结

- `var`：一个循环，一个共享变量。
- `let`：一轮循环，一个独立绑定。
- 闭包捕获绑定，不捕获打印时的字面值。
- 异步回调读变量时，循环通常已经结束。

## 延伸问题

为什么 `for (let i = 0; i < 3; i++)` 中的 `i++` 仍然能影响下一轮？

参考答案：每轮迭代会基于上一轮结束后的值创建新的绑定。不是三轮互不相关的变量，而是每轮都有独立绑定，同时迭代更新会把值传递到下一轮。

## 可能类似的问题及简要参考答案

问：不用 `let`，如何让 `var` 输出 0、1、2？

答：用 IIFE、函数参数、`setTimeout(fn, 0, i)` 的额外参数，或改用 `forEach`，本质都是为每次迭代创建独立绑定。

问：闭包保存的是值还是变量？

答：保存的是变量绑定。变量后续变化时，闭包读取到的是绑定当前的值。

## 辅助记忆总结

用一句话记：`var` 是“三个回调看同一个 i”，`let` 是“三个回调各看各的 i”。
