# 063. [初级]** 数组解构赋值的常见用法

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

数组解构赋值的核心是按位置从数组（或任何可迭代对象）中提取值并绑定到变量。它不是"简写语法"，而是基于迭代器协议的模式匹配：左侧的变量位置对应右侧的迭代顺序。

一句话答法：数组解构按位置取值，支持跳位 `,`、剩余收集 `...rest`、默认值 `= val`、嵌套 `[[a]]` 和函数参数解构，最常见的实战用法是交换变量和解构函数返回的多值数组。

## 问题意图

这道题主要考察三件事：

1. 是否理解数组解构是按位置匹配，而不是按名称匹配（和对象解构的关键区别）。
2. 是否掌握跳位、`...rest`、默认值、嵌套这些常用变体的语法和边界。
3. 是否能将解构用在实际场景中：函数返回值、参数处理、变量交换。

## 考察范围

- 按位置提取：`const [a, b] = arr`，位置对应而非名称对应。
- 跳位提取：用逗号占位跳过不需要的元素，如 `[, , third]`。
- 剩余收集：`...rest` 将剩余元素收集为新数组，必须放在最后。
- 默认值：`[a = 1]` 仅在对应位置的值为 `undefined` 时生效。
- 嵌套数组解构：`const [[x]] = [[1]]`，递归匹配内层数组。
- 函数参数解构：`function fn([a, b]) {}`，在参数位置直接解构。

## 技术错误纠正

- 默认值只在值严格等于 `undefined` 时触发，`null` 不会触发默认值。`const [a = 1] = [null]` 结果是 `a === null`，不是 `1`。
- `...rest` 只能出现在解构模式的最后一个位置，`[...rest, last]` 是语法错误。
- 解构赋值的右侧不限于数组，任何实现了 `Symbol.iterator` 的对象都可以被数组解构，包括字符串、Set、Map、Generator。

## 知识点系统梳理

### 基本语法和位置匹配

```js
const [a, b, c] = [1, 2, 3]
// a = 1, b = 2, c = 3

// 变量多于元素：多出的变量是 undefined
const [x, y] = [1]
// x = 1, y = undefined

// 元素多于变量：多出的元素被忽略
const [m] = [1, 2, 3]
// m = 1
```

### 各模式对比

| 模式 | 语法 | 结果 | 注意事项 |
| --- | --- | --- | --- |
| 基础提取 | `const [a, b] = [1, 2]` | `a=1, b=2` | 按位置，非名称 |
| 跳位 | `const [, , c] = [1, 2, 3]` | `c=3` | 逗号占位 |
| 剩余收集 | `const [a, ...r] = [1, 2, 3]` | `a=1, r=[2,3]` | `...rest` 必须在最后 |
| 默认值 | `const [a = 0] = []` | `a=0` | 仅 `undefined` 触发，`null` 不触发 |
| 嵌套 | `const [[a]] = [[1]]` | `a=1` | 右侧结构必须匹配 |
| 交换变量 | `[a, b] = [b, a]` | 互换 | 无需临时变量 |
| 参数解构 | `([a, b]) => a + b` | 函数参数位置直接解构 | 传入非数组会报错 |

### 和对象解构的区别

| 对比项 | 数组解构 | 对象解构 |
| --- | --- | --- |
| 匹配依据 | 位置（索引顺序） | 属性名 |
| 跳过元素 | 逗号占位 `[, , c]` | 不取即可 |
| 重命名 | 不需要，变量名自由 | `{ name: n }` |
| 适用数据 | 有序序列、元组式返回值 | 配置对象、API 响应 |

### 默认值的触发规则

```js
const [a = 1] = [undefined] // a = 1（undefined 触发默认值）
const [b = 1] = [null]      // b = null（null 不触发）
const [c = 1] = [0]         // c = 0（0 不触发）
const [d = 1] = ['']        // d = ''（空字符串不触发）
```

规则：只有 `undefined`（包括位置不存在）才触发默认值，其他所有 falsy 值都不触发。

## 实战应用举例

### 示例 1：解构函数返回的多值结果

这个例子证明：当函数需要返回多个值时，数组解构比返回对象更简洁，且调用方可以自由命名。

```js
function getMinMax(arr) {
  const sorted = [...arr].sort((a, b) => a - b)
  return [sorted[0], sorted[sorted.length - 1]]
}

const [min, max] = getMinMax([5, 2, 8, 1, 9])
// min = 1, max = 9

// 只需要最大值时，跳过第一个
const [, largest] = getMinMax([3, 7, 2])
// largest = 7
```

边界说明：

- 空数组传入时 `sorted[0]` 为 `undefined`，解构后 `min` 和 `max` 都是 `undefined`。
- 返回数组的位置含义是隐式约定；如果返回值超过 3 个，改用对象解构更清晰。

### 示例 2：在回调参数中解构二维数据

```js
const records = [
  ['Alice', 28, 'engineer'],
  ['Bob', 35, 'designer'],
]

const names = records.map(([name, , role]) => `${name} (${role})`)
// ['Alice (engineer)', 'Bob (designer)']
```

这个例子证明：在 `.map()` 等回调中直接解构，比 `item[0]`、`item[2]` 的下标访问更可读。跳位逗号跳过了不需要的 `age`。

## 使用场景说明和对比

| 场景 | 是否适合数组解构 | 原因 |
| --- | --- | --- |
| 函数返回 2-3 个有序值 | 适合 | 位置含义明确，调用方命名自由 |
| 交换两个变量 | 适合 | `[a, b] = [b, a]` 一行完成 |
| 遍历二维数组提取列 | 适合 | 回调参数解构比下标访问更清晰 |
| 函数返回 4 个以上值 | 不适合 | 位置含义不明确，改用对象解构 |
| API 返回的 JSON 对象 | 不适合 | 数据按名称组织，应用对象解构 |
| 解构长度不确定且需全部处理 | 不适合 | 直接用循环或数组方法更合理 |

## 易错点提示

- `[a, b] = [b, a]` 如果前面有语句且没分号，需要在前面加 `;`，否则解析器会把 `[` 当成上一行的属性访问。
- `const [a = 1] = [null]` 结果是 `null` 不是 `1`；默认值只对 `undefined` 生效。
- `const [...rest, last] = arr` 是语法错误；`...rest` 必须在最后位置。
- 解构不存在的嵌套层级会报错：`const [[a]] = [undefined]` 抛出 TypeError，因为无法对 `undefined` 进行迭代。
- 解构右侧必须是可迭代对象；`const [a] = 123` 会抛出 TypeError。
- 与 `arguments` 混用时注意：`arguments` 是类数组但可迭代，解构可以工作；但普通对象 `{0: 'a', length: 1}` 不可迭代，解构会报错。

## 记忆要点总结

- 数组解构按位置匹配，对象解构按名称匹配。
- 默认值只对 `undefined` 生效，`null`、`0`、`''` 都不触发。
- `...rest` 只能在解构的最后一个位置。
- 返回值超过 3 个就换对象解构，位置含义靠人脑记不住。
- 解构右侧必须是可迭代对象，否则 TypeError。

## 可能类似的问题及简要参考答案

**Q：如何用解构交换两个变量？**
A：`[a, b] = [b, a]`。不需要临时变量，但注意如果前一行没有分号，需要在 `[` 前面加 `;` 防止解析错误。

**Q：解构赋值的默认值什么时候生效？**
A：仅当对应位置的值严格等于 `undefined`（包括位置不存在）时生效。`null`、`0`、`false`、`''` 都不会触发默认值。

**Q：数组解构能用在非数组上吗？**
A：可以。任何实现了 `Symbol.iterator` 接口的对象都支持数组解构，包括字符串、Set、Map、Generator 函数的返回值。
