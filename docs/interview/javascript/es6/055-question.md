# 055. [中级] 箭头函数和普通函数的区别

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

箭头函数和普通函数最大的差异不是写法短，而是运行语义不同：箭头函数没有自己的 `this`、`arguments`、`prototype`，也不能作为构造函数。

一句话回答：箭头函数适合回调和保持外层 `this`；普通函数适合对象方法、构造函数、原型方法和需要动态 `this` 的场景。

## 问题意图

面试官通常想确认你不是“所有函数都改箭头”，而是能按 `this`、构造能力、参数处理和提升行为选择函数类型。

## 考察范围

- `this` 绑定方式。
- 是否有 `arguments`。
- 是否能 `new`。
- 是否有 `prototype`。
- 函数声明提升与函数表达式初始化。
- 对象方法、类方法、事件回调中的取舍。

## 技术错误纠正

`augments` 是拼写错误，应为 `arguments`。

“箭头函数不存在函数提升”要说准确：箭头函数通常是函数表达式，变量声明是否提升取决于 `var/let/const`；但函数值不会像函数声明那样提前初始化。

## 知识点系统梳理

| 对比项 | 箭头函数 | 普通函数 |
| --- | --- | --- |
| `this` | 继承定义位置外层 `this` | 由调用方式决定 |
| `arguments` | 没有自己的 `arguments` | 有自己的类数组 `arguments` |
| `new` | 不能作为构造函数 | 可以作为构造函数 |
| `prototype` | 没有 | 有 |
| 适合场景 | 回调、数组方法、保持外层 `this` | 对象方法、原型方法、构造函数 |
| `call/apply/bind` 改 `this` | 无效 | 有效 |

## 实战应用举例

```js
const user = {
  name: 'Ada',
  sayHi() {
    return `Hi, ${this.name}`
  },
  sayHiArrow: () => `Hi, ${this.name}`,
}

console.log(user.sayHi()) // Hi, Ada
console.log(user.sayHiArrow()) // this 不指向 user
```

对象方法要用普通函数，因为调用时需要 `this` 指向对象本身。

```js
function Timer() {
  this.seconds = 0

  setInterval(() => {
    this.seconds += 1
  }, 1000)
}
```

内部回调用箭头函数是合适的，因为它要继承外层构造函数里的 `this`。

## 使用场景说明和对比

| 需求 | 推荐 | 原因 |
| --- | --- | --- |
| 数组转换、筛选 | 箭头函数 | 简洁，通常不需要自己的 `this` |
| Promise 数据加工 | 箭头函数 | 适合表达链式转换 |
| 对象实例方法 | 普通函数 | 需要动态 `this` |
| 原型方法 | 普通函数 | 需要实例作为 `this` |
| 构造对象 | 普通函数或 class | 箭头函数不能 `new` |
| 可变参数 | rest 参数或普通函数 | 箭头函数没有 `arguments` |

## 易错点提示

- 对象字面量里写箭头函数方法，`this` 往往不是对象本身。
- `bind` 不能改变箭头函数的 `this`。
- 箭头函数没有 `prototype`，不能挂原型方法。
- `const fn = () => {}` 在声明前访问会遇到 TDZ。

## 记忆要点总结

- 箭头函数：短回调、外层 `this`。
- 普通函数：动态 `this`、构造、原型。
- `arguments` 用 rest 参数替代。
- 不要把箭头函数当普通函数的全量替代。

## 延伸问题

为什么对象方法里不建议写箭头函数？

参考答案：对象方法通常希望 `this` 指向调用对象。箭头函数没有自己的 `this`，它继承定义处外层 `this`，所以 `obj.method()` 也不会把 `this` 绑定到 `obj`。

## 可能类似的问题及简要参考答案

问：箭头函数能被 `bind` 改变 `this` 吗？

答：不能。`bind` 可以返回新函数，但箭头函数内部读取的 `this` 仍来自词法作用域。

问：箭头函数有没有 `prototype`？

答：没有，所以不能作为构造函数，也不适合做原型方法。

## 辅助记忆总结

记成一句话：箭头函数看外层上下文，普通函数看调用者。
