# 054. [初级] 箭头函数的基本语法是什么？

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

箭头函数是 ES6 提供的函数表达式简写。它能让回调、数组方法、Promise 链更短，但它不是普通函数的完全替代品。

一句话回答：基本写法是 `(参数) => 表达式` 或 `(参数) => { 语句 }`；单表达式可以隐式返回，代码块必须显式 `return`。

## 问题意图

这题先考语法掌握，再顺手观察你是否知道两个常见边界：返回对象要加括号、箭头函数有词法 `this`。

## 考察范围

- 无参数、单参数、多参数写法。
- 表达式体和代码块体。
- 隐式返回和显式 `return`。
- 返回对象字面量的括号。
- 与普通函数表达式的基础差异。

## 技术错误纠正

无明显已给答案需要纠正。需要补充的是：箭头函数的“简洁”只适用于函数表达式场景，它没有自己的 `this`、`arguments`，不能作为构造函数。

## 知识点系统梳理

| 写法 | 含义 | 示例 |
| --- | --- | --- |
| `() => value` | 无参数，隐式返回 | `() => Date.now()` |
| `x => x * 2` | 单参数可省略括号 | `n => n + 1` |
| `(a, b) => a + b` | 多参数必须加括号 | `(a, b) => a + b` |
| `x => { return x * 2 }` | 代码块体必须显式返回 | `item => { return item.id }` |
| `() => ({ id: 1 })` | 返回对象字面量要加括号 | `name => ({ name })` |

## 实战应用举例

```js
const users = [
  { name: 'Ada', active: true },
  { name: 'Lin', active: false },
]

const activeNames = users.filter(user => user.active).map(user => user.name)

console.log(activeNames) // ['Ada']
```

这个例子证明：箭头函数最适合短回调，读者能直接看到“筛选 active，再取 name”。

```js
const createUser = name => ({ name, role: 'guest' })

console.log(createUser('Ada')) // { name: 'Ada', role: 'guest' }
```

返回对象时必须写成 `({ ... })`。否则 `{}` 会被解析成函数体，不是对象字面量。

## 使用场景说明和对比

| 场景 | 箭头函数是否适合 | 原因 |
| --- | --- | --- |
| `map/filter/reduce` 短回调 | 适合 | 语义短，减少样板代码 |
| Promise 链 | 适合 | 适合表达数据转换 |
| 对象方法 | 谨慎 | `this` 不指向调用对象 |
| 构造函数 | 不适合 | 不能被 `new` 调用 |
| 需要 `arguments` | 不适合 | 应改用 rest 参数 |

## 易错点提示

- 单参数可以省略括号，但无参数和多参数不能省。
- 代码块体不会自动返回，必须写 `return`。
- 返回对象字面量要用括号包起来。
- 箭头函数是表达式，通常赋给变量，不是函数声明。

## 记忆要点总结

- `参数 => 结果` 是最小形态。
- 花括号表示函数体，不表示对象返回。
- 短回调用箭头函数最清晰。
- 需要动态 `this` 时不要用箭头函数。

## 延伸问题

为什么 `const fn = () => ({ id: 1 })` 要给对象加括号？

参考答案：箭头后面的 `{}` 默认按函数体解析。加括号后，解析器才会把它当作表达式里的对象字面量。

## 可能类似的问题及简要参考答案

问：箭头函数能不能写多行？

答：可以，用 `{}` 包住函数体，但需要显式 `return`。

问：箭头函数和匿名函数表达式有什么关系？

答：箭头函数是一种更短的函数表达式，但 `this`、`arguments`、构造能力不同。

## 辅助记忆总结

记成一句话：箭头函数适合“短参数到短结果”，一旦需要自己的 `this` 或构造能力，就换普通函数。
