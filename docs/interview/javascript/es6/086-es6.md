# 086. [初级] ES6 中如何定义一个类？

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

ES6 的 `class` 是原型继承的语法糖，不是新的面向对象模型。面试问"如何定义类"是考察你是否能用 `class` 关键字写出包含构造函数、实例方法、静态成员的基础结构，并理解它和 ES5 构造函数的本质联系。

一句话答法：用 `class` 关键字定义，`constructor` 做初始化，方法直接写在类体内；`class` 本质还是函数，不存在真正的"类"。

## 问题意图

1. 是否能写出 `class` 的基本骨架（constructor + 实例方法）。
2. 是否知道 `class` 是语法糖，底层仍是原型链。
3. 是否了解 class 的特殊行为：不提升、严格模式、不可枚举、必须配合 `new`。

## 考察范围

- `class` 关键字基本语法。
- `constructor` 的作用和唯一性。
- 实例方法定义（不加 `function` 关键字）。
- 静态方法（`static`）和静态属性。
- getter/setter（`get` / `set`）。
- class 声明 vs ES5 构造函数的关系。
- class 的特性：不提升、严格模式、方法不可枚举、`[[IsClass]]` 内部标记。

## 技术错误纠正

- 原始代码中 `constructor` 内属性赋值是正确的，但注意 class 内部默认严格模式，未声明变量会报错。
- `class` 的方法默认是不可枚举的，这和 ES5 原型方法可枚举不同。

## 知识点系统梳理

### 基本语法

```js
class Student {
  constructor(name, age) {
    this.name = name
    this.age = age
  }

  // 实例方法（挂到 Student.prototype 上）
  introduce() {
    return `Hi, I'm ${this.name}`
  }

  // 静态方法（挂到 Student 本身上）
  static createGuest() {
    return new Student('Guest', 0)
  }

  // Getter
  get info() {
    return `${this.name}, ${this.age}`
  }

  // Setter
  set studentAge(val) {
    if (val > 0) this.age = val
  }
}
```

### class 与 ES5 构造函数对照

| 对比项 | ES5 构造函数 | ES6 class |
| --- | --- | --- |
| 定义方式 | `function Person(name) {}` | `class Person { constructor(name) {} }` |
| 实例方法 | `Person.prototype.say = fn` | 直接写 `say() {}` 在类体内 |
| 静态方法 | `Person.create = fn` | `static create() {}` |
| 调用约束 | 可被普通调用 `Person()` | 必须 `new Person()`，否则报错 |
| 提升 | 声明提升 | 不提升（TDZ） |
| 方法可枚举性 | 原型方法可枚举 | 方法不可枚举 |
| 严格模式 | 非严格模式（默认） | 内部始终严格模式 |

### class 的本质

```js
typeof Student // "function"
Student === Student.prototype.constructor // true
```

`class` 只是 `function` 的另一种写法，底层仍是构造函数 + 原型赋值。

## 实战应用举例

### 示例 1：基础数据模型

```js
class User {
  constructor(data = {}) {
    this.id = data.id
    this.name = data.name
    this.email = data.email
  }

  toJSON() {
    return { id: this.id, name: this.name, email: this.email }
  }

  static fromJSON(json) {
    return new User(json)
  }
}
```

### 示例 2：静态工厂方法

```js
class Logger {
  static info(msg) { console.log(`[INFO] ${msg}`) }
  static warn(msg) { console.warn(`[WARN] ${msg}`) }
  static error(msg) { console.error(`[ERROR] ${msg}`) }
}

// 无需实例化
Logger.info('App started')
```

## 使用场景说明和对比

| 场景 | 是否适合 class | 原因 |
| --- | --- | --- |
| 数据模型（User、Product） | 适合 | 需要统一初始化和方法 |
| 工具类静态方法集合 | 适合 | 无需实例化，静态方法直接调用 |
| 轻量数据容器（DTO） | 通常不适合 | 普通对象或 POJO 更简单 |
| 函数工具库 | 不适合 | 直接导出函数更轻量 |

## 易错点提示

- `class` 不会提升：`new Foo()` 写在 `class Foo {}` 之前会 `ReferenceError`。
- class 必须用 `new` 调用：`Student()` 报 `TypeError: Class constructor Student cannot be invoked without 'new'`。
- class 方法不可枚举，`for...in` 遍历不到。
- class 体内方法间不需要逗号或分号分隔，直接在下一行写即可。

## 记忆要点总结

- `class` 是构造函数的语法糖，`typeof` 返回 `"function"`。
- 三大块：constructor（初始化）、实例方法（原型）、静态方法（类本身）。
- 三特性：不提升、严格模式、必须 `new`。

## 可能类似的问题及简要参考答案

**Q：ES6 class 和 ES5 构造函数是什么关系？**
A：`class` 是语法糖，本质仍是构造函数。`typeof MyClass === 'function'`。

**Q：class 会提升吗？**
A：不会。`class` 存在暂时性死区（TDZ），必须先声明后使用。

**Q：class 可以不写 constructor 吗？**
A：可以。不写时引擎会自动添加一个空的 `constructor()`。

**Q：class 中能调用非 `new` 的 class 吗？**
A：不能。class 内部机制会检查 `[[IsClass]]` 标记，普通调用会抛 TypeError。

## 辅助记忆总结

记成一句话：`class` = `function` + `prototype` 的语法糖，区别是不提升、严格模式、必须 `new`。
