# 059. [中级] 什么时候不应该使用箭头函数，什么时候应该使用箭头函数？

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

选择箭头函数的关键不是“短不短”，而是是否需要自己的 `this`、`arguments`、`prototype` 或构造能力。

一句话回答：需要继承外层 `this` 或写短回调用箭头函数；需要调用者决定 `this`、需要构造、需要原型、需要 DOM 元素 `this` 时用普通函数。

## 问题意图

这题考察工程判断。面试官通常会给你对象方法、事件监听、Vue methods、数组回调，让你判断箭头函数会不会改变语义。

## 考察范围

- 词法 `this` 和动态 `this`。
- 构造函数、原型方法。
- DOM 事件和框架事件。
- 回调函数、数组方法、Promise 链。
- rest 参数替代 `arguments`。

## 技术错误纠正

“类的方法不应该使用箭头函数”要分场景。原型上的 class method 通常用普通方法；如果是 class field 形式的实例回调，箭头函数可以用来绑定实例 `this`，但会为每个实例创建一份函数。

## 知识点系统梳理

| 判断问题 | 用箭头函数 | 用普通函数 |
| --- | --- | --- |
| 是否需要外层 `this` | 是 | 否 |
| 是否需要调用者作为 `this` | 否 | 是 |
| 是否需要 `new` | 否 | 是 |
| 是否要挂到原型上 | 否 | 是 |
| 是否只是短回调 | 是 | 也可以，但较啰嗦 |
| 是否需要 `arguments` | 用 rest | 可以用 `arguments` |

## 实战应用举例

不应该用箭头函数：

```js
const user = {
  name: 'Ada',
  getName: () => this.name,
}

console.log(user.getName()) // 不是 Ada
```

应该用普通方法：

```js
const user = {
  name: 'Ada',
  getName() {
    return this.name
  },
}
```

适合用箭头函数：

```js
const prices = [10, 20, 30]
const withTax = prices.map(price => price * 1.1)
```

这个回调不需要自己的 `this`，箭头函数最清楚。

## 使用场景说明和对比

| 场景 | 推荐 | 原因 |
| --- | --- | --- |
| 数组方法回调 | 箭头函数 | 表达转换关系 |
| Promise 链 | 箭头函数 | 减少样板代码 |
| 方法内部定时器 | 箭头函数 | 保留外层实例 |
| 对象方法 | 普通函数 | `this` 应指向对象 |
| 原型方法 | 普通函数 | `this` 应指向实例 |
| Vue `methods` | 普通函数 | Vue 需要绑定组件实例 |
| DOM 事件需要元素 `this` | 普通函数 | 浏览器会设置监听器 `this` |

## 易错点提示

- 事件处理器不是一律不能用箭头函数；如果用 `event.currentTarget`，箭头函数也可以。
- 对象方法里用箭头函数通常是错的。
- class field 箭头函数能绑定实例，但会牺牲原型方法共享。
- 需要动态 `this` 时，`bind` + 普通函数比箭头函数更明确。

## 记忆要点总结

- 看 `this`：要外层，用箭头；要调用者，用普通函数。
- 看构造：要 `new`，不用箭头。
- 看原型：要共享方法，用普通函数。
- 看短回调：优先箭头。

## 延伸问题

React 类组件里为什么常见箭头函数类字段？

参考答案：它能让事件回调稳定绑定组件实例，避免在构造函数里手动 `bind`。代价是函数挂在实例上，而不是原型上。

## 可能类似的问题及简要参考答案

问：Vue `methods` 能写箭头函数吗？

答：不建议。Vue 会把普通方法的 `this` 绑定到组件实例，箭头函数会跳过这个绑定。

问：DOM 事件中箭头函数一定错吗？

答：不一定。如果只用 `event.target/currentTarget`，可以用；如果依赖监听器里的 `this` 指向元素，就要用普通函数。

## 辅助记忆总结

记成一句话：箭头函数适合“借外层”，普通函数适合“看调用”。
