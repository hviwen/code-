# 057. [初级] 箭头函数可以作为构造函数吗？

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

箭头函数不能作为构造函数。它没有 `[[Construct]]` 内部方法，也没有 `prototype` 属性，使用 `new` 调用会抛 `TypeError`。

一句话回答：不能，创建实例请用 `class` 或普通函数。

## 问题意图

这题考察你是否理解箭头函数的设计目的：它是轻量函数表达式和回调语法，不是对象实例化机制。

## 考察范围

- `new` 调用需要构造能力。
- `[[Construct]]` 内部方法。
- `prototype` 属性。
- 箭头函数词法 `this`。
- `class`、普通构造函数、工厂函数的替代关系。

## 技术错误纠正

原答案“不可以”结论正确，但不够复习。需要说明原因：箭头函数没有构造调用能力，没有自己的 `this`，也没有可用于实例原型链的 `prototype`。

## 知识点系统梳理

| 能力 | 箭头函数 | 普通函数 | class |
| --- | --- | --- | --- |
| `new` 调用 | 不支持 | 支持 | 支持 |
| `prototype` | 无 | 有 | 有 |
| 自己的 `this` | 无 | 有 | 构造时绑定实例 |
| 适合用途 | 回调、表达式 | 方法、构造、普通逻辑 | 对象建模 |

构造函数至少要能做两件事：创建新对象，并把 `this` 指向这个新对象。箭头函数两者都不承担。

## 实战应用举例

```js
const User = name => {
  this.name = name
}

new User('Ada') // TypeError: User is not a constructor
```

正确替代：

```js
class User {
  constructor(name) {
    this.name = name
  }
}

const user = new User('Ada')
console.log(user.name) // Ada
```

如果不需要原型和 `instanceof`，可以用工厂函数：

```js
const createUser = name => ({
  name,
  sayHi() {
    return `Hi, ${this.name}`
  },
})
```

## 使用场景说明和对比

| 需求 | 推荐方案 | 原因 |
| --- | --- | --- |
| 需要实例和原型方法 | `class` | 语义清晰，支持 `new` |
| 维护老构造函数 | 普通函数 | 兼容原型写法 |
| 只返回普通对象 | 工厂函数 | 不需要 `new` |
| 写短回调 | 箭头函数 | 简洁，不需要构造能力 |

## 易错点提示

- 箭头函数没有 `prototype`，`Arrow.prototype` 是 `undefined`。
- 给箭头函数加大写命名也不能让它变成构造函数。
- `new` 箭头函数报错，不是返回空对象。
- 工厂函数可以是箭头函数，因为它不是被 `new` 调用。

## 记忆要点总结

- 箭头函数不能 `new`。
- 没有 `prototype`，没有构造能力。
- 建实例用 `class` 或普通函数。
- 返回对象可用箭头工厂函数。

## 延伸问题

为什么工厂函数可以用箭头函数，而构造函数不可以？

参考答案：工厂函数只是普通调用并返回对象，不需要 `new`、`prototype` 或实例 `this`。构造函数依赖 `new` 创建实例，箭头函数不支持这个机制。

## 可能类似的问题及简要参考答案

问：箭头函数有没有 `prototype`？

答：没有，所以不能挂原型方法，也不能用于构造实例。

问：`class` 本质上和普通构造函数有什么关系？

答：`class` 是基于原型机制的语法形式，能被 `new` 调用，并把方法放到原型上。

## 辅助记忆总结

记成一句话：箭头函数负责“算结果”，不负责“造对象”。
