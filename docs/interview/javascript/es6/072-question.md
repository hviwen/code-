# 072. [初级]** 什么是模板字符串？如何使用？

> 来源：`docs/javascript/js_interview_questions_part_2.md`

## 问题本质解读

模板字符串（template literal）是 ES6 引入的字符串字面量语法，用反引号 `` ` `` 包围，支持 `${}` 嵌入表达式、原生多行和标签函数处理。它替代了传统的 `+` 拼接，让字符串组装更直观。

一句话答法：模板字符串用反引号定义，`${expression}` 插值，原生支持多行，比 `+` 拼接可读性更强。

## 问题意图

这道题主要考察三件事：

1. 是否掌握反引号 + `${}` 的基本语法，能和普通引号区分开。
2. 是否知道 `${}` 里可以放任意表达式（不仅是变量），以及多行不需要 `\n`。
3. 是否了解标签模板（tagged template）的存在和用途。

## 考察范围

- 反引号 `` ` `` 语法与普通引号 `'` `"` 的区别。
- `${}` 插值：变量、表达式、函数调用。
- 多行字符串：不需要 `\n` 或 `+` 拼接。
- `${}` 内部的表达式求值规则。
- 与 `+` 拼接的可读性和维护性对比。
- 标签模板（tagged template）的基本概念。

## 技术错误纠正

- 原始材料中 `Math.max(arr)` 和 `Math.min(arr)` 应使用展开运算符：`Math.max(...arr)`。
- `${}` 内部是**表达式**，不是语句；不能写 `if`、`for` 等语句，但可以用三元表达式。
- 模板字符串内的换行是真实换行符（`\n`），缩进空格也会保留，生成 HTML 时要注意多余空白。

## 知识点系统梳理

### 基本语法

```js
const name = 'Alice'
const greeting = `Hello, ${name}!` // "Hello, Alice!"
```

`${}` 内部可以是任意表达式：

```js
`${1 + 2}`             // "3"
`${ok ? 'yes' : 'no'}` // 三元表达式
`${arr.join(', ')}`     // 方法调用
`${fn()}`               // 函数调用
```

### 多行字符串

```js
// 传统写法
const old = '第一行\n' +
  '第二行\n' +
  '第三行'

// 模板字符串：直接换行
const tpl = `第一行
第二行
第三行`
```

注意：模板字符串中的缩进空格和换行都会保留在结果中。

### 模板字符串 vs 普通字符串拼接

| 对比项 | `+` 拼接 | 模板字符串 |
| --- | --- | --- |
| 语法 | `'Hello, ' + name + '!'` | `` `Hello, ${name}!` `` |
| 多行 | 需要 `\n` 或 `+` 换行 | 直接换行 |
| 可读性 | 变量多时引号和 `+` 容易混乱 | 结构一目了然 |
| 表达式 | 需要括号辅助阅读 | `${}` 自然包裹 |
| 性能 | 现代引擎无显著差异 | 现代引擎无显著差异 |
| 兼容性 | 所有环境 | ES6+（IE 不支持） |

### 标签模板简介

标签模板是在模板字符串前加一个函数名，函数会接收拆分后的字符串片段和插值结果：

```js
function highlight(strings, ...values) {
  return strings.reduce((result, str, i) =>
    result + str + (values[i] !== undefined ? `<mark>${values[i]}</mark>` : ''), '')
}

const name = 'Alice'
highlight`Hello, ${name}!` // "Hello, <mark>Alice</mark>!"
```

常见应用：`styled-components` 的 CSS 模板、SQL 防注入的 `sql` 标签、国际化处理。

## 实战应用举例

### 示例 1：HTML 片段拼装

这个例子展示模板字符串在动态生成 HTML 时比 `+` 拼接清晰得多，同时说明缩进空白的影响。

```js
function renderCard(user) {
  return `<div class="card">
  <h2>${user.name}</h2>
  <p>Age: ${user.age}, Status: ${user.active ? 'Active' : 'Inactive'}</p>
</div>`
}

renderCard({ name: 'Bob', age: 30, active: true })
// 输出：
// <div class="card">
//   <h2>Bob</h2>
//   <p>Age: 30, Status: Active</p>
// </div>
```

边界说明：

- `user.name` 如果包含 `<script>` 等内容，直接插值会导致 XSS。生产环境要转义或使用框架的模板引擎。
- 模板内的缩进空格会出现在最终字符串里。如果需要无缩进的 HTML，要么顶格写，要么用 `.trim()` + 去缩进工具。

### 示例 2：日志格式化

```js
function log(level, msg, meta = {}) {
  const ts = new Date().toISOString()
  const extra = Object.keys(meta).length ? ` | ${JSON.stringify(meta)}` : ''
  return `[${ts}] ${level.toUpperCase()}: ${msg}${extra}`
}

log('error', 'DB connection failed', { host: 'localhost', retries: 3 })
// "[2026-06-25T10:00:00.000Z] ERROR: DB connection failed | {\"host\":\"localhost\",\"retries\":3}"
```

比 `'[' + ts + '] ' + level.toUpperCase() + ': ' + msg + extra` 可读性强得多。

## 使用场景说明和对比

| 场景 | 是否适合模板字符串 | 原因 |
| --- | --- | --- |
| 拼接 2 个以上变量的字符串 | 适合 | 比 `+` 拼接更清晰 |
| 多行 HTML/SQL/日志模板 | 适合 | 原生换行，无需 `\n` |
| 只拼一个变量的简单字符串 | 都可以 | `'Hi, ' + name` 和 `` `Hi, ${name}` `` 差异不大 |
| 需要防 XSS / SQL 注入 | 配合标签模板 | 标签函数可以对插值做转义 |
| 纯静态字符串，无变量 | 不需要 | 普通引号更简洁 |
| 需要兼容 IE | 不适合 | IE 不支持模板字符串，需要 Babel 编译 |

## 易错点提示

- **反引号 vs 单引号**：`` ` `` 和 `'` 外观相似，在某些字体下容易混淆。模板字符串必须用反引号，普通引号内 `${}` 不会被解析。
- **`${}` 只能放表达式**：`${if (x) {...}}` 会报语法错误，应改为 `${x ? a : b}` 三元表达式。
- **缩进空白会保留**：函数内缩进的模板字符串，缩进空格会出现在输出中，可能影响 HTML 渲染或日志格式。
- **嵌套反引号需转义**：模板字符串内使用反引号要写 `` \` ``，或者用标签模板的 `String.raw`。
- **`undefined` 和 `null` 会直接输出字符串**：`${undefined}` 输出 `"undefined"`，不会报错也不会输出空字符串。
- **XSS 风险**：直接把用户输入插值到 HTML 模板字符串里和 `innerHTML` 一样危险，不等于"安全的模板引擎"。

## 记忆要点总结

- 反引号 `` ` `` 定义，`${}` 插值，直接换行支持多行。
- `${}` 内只能是表达式，不能写语句。
- 缩进和换行都会保留在结果字符串中。
- `${undefined}` 输出 `"undefined"`，不是空串。
- 标签模板可以自定义插值处理逻辑（转义、高亮、国际化）。

## 可能类似的问题及简要参考答案

**Q：模板字符串可以嵌套吗？**
A：可以。`${}` 内部可以再写模板字符串，例如 `` `${`inner ${x}`}` ``，但嵌套过多会降低可读性。

**Q：模板字符串和字符串拼接性能哪个更好？**
A：现代引擎两者性能基本一致，选择模板字符串是为了可读性，不是为了性能。

**Q：如何在模板字符串中输出反引号本身？**
A：使用反斜杠转义：`` `这是一个反引号：\`` ``。

## 辅助记忆总结

记成一句话：模板字符串就是"反引号包围 + `${}` 插表达式 + 原生换行"，比 `+` 拼接更清晰，标签模板可自定义处理。
